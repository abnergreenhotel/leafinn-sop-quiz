$CATEGORY: OP-FO-26 顧客訊息回覆

::OP-FO-26-Q01::當您在處理 E-MAIL 時，收到一份來自總部的公文或需經由主管審核的信件，根據標準作業流程（SOP），您應該如何處理？ {
~直接依照公文指示內容回覆寄件者，並於結案後彙整至月報表。#SOP 明確規定公文或總部信件必須先轉給主管審閱，而非直接回覆。
=將該信件轉給主管審閱，確認後續處理方式。#根據作業指導書，總部信件、公文及不知職權歸屬之信件，皆須轉給主管審閱。
~先回覆寄件者表示已收到信件，再自行判斷內容分發至相關部門。#對於公文類的特殊文件，SOP 強調的是主管審閱權，而非第一線人員的判斷分發。
~暫緩處理，等待主管每日例行巡檢信箱時自行查看。#這會違反 30 分鐘內須回覆一次訊息的時效規定，應主動轉達。
~轉傳給行銷部，由行銷部統一彙整後呈報總部。#行銷部僅負責行銷、企業合作或抽獎相關事宜，一般公文應轉給直屬主管。
}

::OP-FO-26-Q02::在處理顧客訊息回覆時，針對「回覆時效」的標準要求為何？ {
~須在收件後的 10 分鐘內完成最終處理並結案。#SOP 要求的是「初次」回覆或回應的時效，而非要求在極短時間內完全結案。
~僅針對訂房類訊息須於 15 分鐘內回覆，其餘信件可於當日回畢。#30 分鐘的回覆標準適用於所有信件與顧客訊息，並無類別之分。
=須於 30 分鐘內回覆一次信件及顧客訊息。#作業指導書明確規定，不論是 E-MAIL 或 FB/LINE，基本回覆標準皆為 30 分鐘內。
~若訊息過多，可延至 60 分鐘內統一批次處理。#SOP 訂定的時效標準為 30 分鐘，以確保顧客滿意度與即時性。
~只需在下班前確認當日訊息皆已回覆完畢即可。#這與「30 分鐘內回覆一次」的時效規範嚴重不符。
}

::OP-FO-26-Q03::處理 OTA 訂單信件時，若發現訂單內註明了特殊需求（例如：需要澡盆），下列哪一項操作符合 SOP 流程？ {
~僅需在 OTA 平台後台回覆顧客「沒問題」即可。#除了回覆顧客，還需在內部系統完成備註與庫存控管的動作。
=至德安前台找到訂單加上備註，並於備註處註明扣除備品數量。#這是為了確保現場同仁能看到需求，並即時更新備品庫存（需扣除數量）。
~直接將需求通知房務部門，不需在德安系統進行任何操作。#SOP 要求必須至德安前台系統找到訂單進行備註，以維持資料一致性。
~若備品足夠則不需做任何動作，若不足才需回覆顧客。#不論數量多寡，只要有特殊需求皆應在系統內註記以供查閱。
~在 WEB 後台點選「已收到需求」，系統會自動同步至德安系統，不需人工操作。#SOP 指出特殊需求需人工至德安前台進行備註與備品數量扣除。
}

::OP-FO-26-Q04::在處理 OTA 串單異常時，若發現「無串單但不可下傳」的訂單，正確的後續處理步驟為何？ {
~直接在 WEB 後台按「下傳」並等待五分鐘。#若訂單已經被標記為「不可下傳」，重複執行下傳動作通常無效。
~通知顧客該訂單無效，請顧客重新下單。#這會造成顧客困擾，並非處理系統異常的正確流程。
=可手建新單，處理完畢後回報主管。#當系統無法自動下傳且確認無串單時，應手動建立訂單並向主管核備。
~等待系統每小時自動修復，不需人工介入。#異常訂單需即時處理，不可消極等待系統自動修復。
~直接在德安系統修改其他現有訂單的資料覆蓋上去。#嚴禁覆蓋現有訂單資料，這會造成帳務與資訊混亂。
}

::OP-FO-26-Q05::若在處理 FB 或 LINE 訊息時收到「客訴」，根據作業指導書，第一線人員應採取什麼行動？ {
~先代表公司致歉，並承諾給予下次入住優惠以平息顧客怒火。#第一線人員不應隨意給予承諾，客訴依規須轉給主管處理。
~將訊息轉給行銷部，由行銷部進行公關危機處理。#SOP 規定客訴應直接轉給主管，而非行銷部（行銷部負責抽獎、合作等）。
=將該客訴訊息轉交給主管處理。#FB/LINE 的客訴處理標準程序即為「客訴需轉給主管」。
~請顧客改寄 E-MAIL 以留下正式紀錄，不建議在社群平台回覆。#SOP 雖要求轉給主管，但未要求引導顧客轉換平台，且需維持 30 分鐘回覆率。
~自行回覆初步解釋，若顧客仍不滿意再轉交主管。#為求慎重，SOP 要求客訴發生時應直接轉給主管審核或處理。
}

::OP-FO-26-Q06::當有顧客透過通訊軟體詢問「大量訂房」或是「常住優惠」時，同仁應如何處置？ {
~告知顧客目前所有房價皆以官網顯示為準，無額外優惠。#對於特殊訂房需求，應有彈性調整空間，不應直接回絕。
~請顧客直接向行銷部洽談企業合作方案。#行銷部處理的是合作開發，一般的訂房優惠仍建議先由主管判斷。
~自行計算 9 折優惠價回覆顧客，以爭取訂單。#第一線人員無權自行決定優惠折扣，須遵循權責劃分。
=可向主管洽詢後再行回覆顧客。#SOP 規定大量訂房或常住之優惠可洽詢主管，以決定適當的價格方案。
~建議顧客改至 OTA 平台訂房，利用平台的紅利點數抵扣。#這不是處理大量訂房需求的正軌流程，且會增加公司的佣金支出。
}

::OP-FO-26-Q07::在處理 OTA 訂單時，若發現顧客要求的備品（如：床圍）目前已無庫存可提供，應採取何種即時措施？ {
~等顧客辦理入住（Check-in）時再由櫃檯人員當面致歉並解釋。#這會導致顧客期望落差，應儘早告知。
=需第一時間在後台告知顧客。#作業指導書注意事項明確規定，若訂單已無顧客要求之備品，需第一時間在後台告知顧客。
~自行去採購新的備品，確保顧客能使用。#採購不屬於訊息回覆的 SOP 範疇，應優先執行通報義務。
~不需主動聯繫，若顧客主動追蹤再告知已無庫存。#這屬於消極處理，不符合服務標準作業指導書的服務品質要求。
~直接取消該筆訂單，因為無法滿足顧客需求。#無法提供單一備品不應直接取消訂單，應與顧客溝通。
}

::OP-FO-26-Q08::關於「評論回覆」（如 AGODA、GOOGLE 等）的操作要求，下列敘述何者錯誤？ {
~每日都需確認各平台是否有新評論。#這是 SOP 中的明確要求，並非錯誤敘述。
~需將評論內容登記在顧客意見表。#這是行政紀錄流程的一環，為正確流程。
~回覆內容需注意有無錯字及注意用詞。#文字品質是回覆評論的基本要求，為正確流程。
~回覆時可參考其他夥伴過去的回覆內容。#這是 SOP 建議的參考方式，可確保回覆的一致性。
=負面評論可選擇不予回覆，僅需登記在意見表即可。#SOP 要求回覆內容需注意用詞，並未指示可以忽略負面評論；所有評論均應專業回覆。
}

::OP-FO-26-Q09::當您在 WEB 後台處理串單異常，並點選了「下傳」按鈕後，後續的標準動作為何？ {
~立即刷新網頁，確認訂單是否已出現。#系統處理需要時間，立即刷新可能無法看到更新後的結果。
=等待五分鐘，再確認訂單正確下傳及資料無誤。#SOP 指出下傳後應等待五分鐘，以確保系統同步程序完成。
~直接在德安系統手動輸入訂單，以防系統延遲。#應先等待系統自動下傳，若五分鐘後仍失敗才考慮其他人工處理方案。
~立刻回報主管下傳動作已執行。#應先確認下傳是否成功及資料是否正確後，若有異常才需回報。
~持續點擊「下傳」按鈕直到德安系統出現訂單為止。#重複點擊可能導致系統產生重複訂單或報錯。
}

::OP-FO-26-Q10::若同仁收到關於「行銷抽獎」或「企業合作」相關的問題，應將訊息轉傳至哪一個單位？ {
~總部行政處#總部處理的是公文審閱，具體的行銷事務有專門部門。
~訂房組長#訂房組僅負責一般訂房流程的導流。
=行銷部#根據 SOP，行銷、抽獎、企業合作等相關問題須轉傳給行銷部處理。
~前台值班經理#值班經理負責現場營運，非行銷合作的主要對接窗口。
~財務部#財務部不負責第一線的行銷諮詢回覆。
}
