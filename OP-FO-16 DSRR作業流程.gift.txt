$CATEGORY: OP-FO-16 DSRR作業流程

::OP-FO-16-Q01::在 DSRR 作業流程中，「DSRR」代表的完整英文意涵為何？ {
~Daily Suite Room Rate#這是一個常見的旅館業術語，但並非此作業流程中所指的隱藏房價代號。
=Don't Show Room Rate#根據文件標題與內容，DSRR 指的是「不顯示房間價格」，即飯店不向入住客人揭露其訂房成本。
~Discount Special Room Rate#雖然 DSRR 訂單通常與優惠或合約價有關，但其核心規範在於「不顯示」而非「折扣」。
~Double Stay Room Rate#此為完全不相關的造詞，與隱藏房價的作業流程無關。
~Direct Sales Room Rate#DSRR 往往涉及第三方代訂或旅行社，而非僅限於直接銷售的房價。
}

::OP-FO-16-Q02::當客人表示房費是由友人代訂（訂房人代訂），且在櫃檯詢問房價時，夥伴應採取何種處理方式？ {
~直接告知客人該訂單的合約房價#這違反了 DSRR 的核心規範，且可能洩露代訂人的隱私或合約機密。
=委婉請客人詢問代訂人#作業流程規範，對於代訂類訂單應引導客人直接與其代訂者溝通以獲取價格資訊。
~告知客人現場房價以供參考#隨意告知現場房價可能會導致客人與代訂人間的價格誤會或糾紛。
~拒絕提供任何住宿相關資訊#我們仍需提供住宿服務，只是在房價資訊上需進行控管，而非全面拒絕溝通。
~要求客人出示代訂人的授權書才可告知#流程中並無此要求，標準做法是請客人直接與代訂人確認。
}

::OP-FO-16-Q03::針對「旅行社」訂單的客人詢問房價，下列哪一項話術或作業規定是正確的？ {
~詳細解釋旅行社與飯店之間的佣金抽成比例#飯店與旅行社的商業合約細節不應對客人揭露。
=委婉告知飯店端沒辦法提供房價資訊#作業規範要求對旅行社訂單應明確表示無法提供，以維護合約神祕性。
~主動告知客人當日的現場房價（Walk-in Rate）#執行說明中明確提到「盡量也不要告知現場房價」，以避免產生比價心理。
~建議客人直接取消訂單改為現場訂房#這會損害合作夥伴（旅行社）的利益，且不符合標準服務流程。
~告知客人旅行社的訂房金額通常比較貴#這種說法缺乏事實根據且極不專業，容易引發客人不滿。
}

::OP-FO-16-Q04::當處理 OTA（線上訂房平台）訂單時，若客人詢問訂房金額，櫃檯夥伴應如何說明？ {
~告知因為系統連線問題，現在查不到#這並非誠實的說法，標準話術應聚焦於權限或系統顯示限制。
~告知客人我們只能看到稅前金額#這仍揭露了部分房價資訊，不符合 DSRR 的精神。
=委婉告知我們在系統上看不到訂房金額#這是執行說明中針對 OTA 訂單指定的標準話術，用以回應價格查詢。
~請客人直接撥打 OTA 平台的客服電話詢問#雖然是方法之一，但標準首選應是告知飯店端無法看到金額。
~告知客人飯店會另外寄送電子發票至其信箱#OTA 訂單通常由平台開立憑證，飯店不一定能直接寄送包含房費的發票。
}

::OP-FO-16-Q05::根據 2026/04/09 修訂的注意事項，關於「住宿證明」的開立限制為何？ {
=住宿證明不可開立房費項目#這是文件中特別強調的新增注意事項，旨在防止 DSRR 價格透過證明文件外流。
~住宿證明必須包含完整的稅金明細#若不可開立房費，自然也不會包含與房費相關的稅金明細。
~住宿證明僅限於入住當天開立#文件中並無針對開立時間的限制，而是針對「內容項目」進行限制。
~住宿證明必須由經理簽核後方可顯示金額#規範中是直接禁止開立房費，並非透過簽核即可顯示。
~住宿證明只能提供給代訂人，不能給房客#房客可以領取證明，但證明上的內容（房費）必須受到限制。
}

::OP-FO-16-Q06::在處理旅行社或 OTA 客人的房價問題時，為何指導書建議「盡量不要告知現場房價」？ {
~因為現場房價每日浮動，容易講錯#雖然房價會浮動，但主要原因並非怕講錯，而是預防價格爭議。
~為了節省辦理入住手續的時間#告知房價與否對手續時間的影響微乎其微。
=避免客人將現場房價與其訂房成本做比較而產生後續困擾#當客人發現兩者有價差時，常會向櫃檯或旅行社要求退費或補償，增加處理成本。
~因為現場房價屬於商業機密，不可對外公開#現場房價通常是公開透明的，並非機密，只是在特定情境下不宜主動揭露。
~為了強迫客人以後都透過旅行社訂房#飯店通常更歡迎直接訂房，此說法完全不符邏輯。
}

::OP-FO-16-Q07::一位 OTA 客人堅持要求在「住宿證明」上註明他支付給平台的 $3,500 元房費，你應如何處理？ {
~既然客人知道金額，就直接幫他寫在證明上#這違反了「住宿證明不可開立房費」的硬性規定。
=委婉告知公司規定住宿證明不可開立房費項目#應堅持標準作業程序，不因客人知道金額就破例開立。
~請客人在住宿證明上自己手寫金額，飯店蓋章證明#這屬於偽造或不實證明的範疇，具有法律風險。
~請客人提供 OTA 的收據，我們根據該收據開立飯店發票#飯店不能針對非由飯店收取的款項開立發票或正式證明。
~告知客人若要顯示金額，需額外支付 10% 的手續費#這是不正當的收費要求，且不符合文件規範。
}

::OP-FO-16-Q09::在執行 DSRR 流程時，「委婉」一詞在話術中多次出現，這代表什麼溝通核心？ {
~代表語氣要卑微，不斷向客人道歉#專業服務不代表卑微，而是以得體且尊重的態度說明。
~代表要避重就輕，轉移客人的注意力#「委婉」是為了維持服務品質並堅定立場，而非逃避問題。
=在不揭露限制資訊的前提下，給予客人合適的引導與建議#委婉地拒絕或解釋，是為了在執行規定的同時，兼顧客人的感受。
~代表語速要放慢，確保客人聽得清楚#委婉與語速無關，與措辭與態度較有關聯。
~代表只能使用氣聲說話，避免被其他客人聽到金額#這並非委婉的定義，且溝通應保持正常且專業的音量。
}

::OP-FO-16-Q10::若一位代訂房客人（DSRR）在現場要求延長住宿並詢問續住價格，根據流程邏輯應如何處理最適當？ {
~直接以當初代訂的低價讓客人續住#原本的代訂價格通常是特定合約價，續住應重新評估房價。
=請客人再次聯絡原代訂人處理續住事宜#這符合「委婉請客人詢問代訂人」的流程邏輯，維持價格的一致性與隱密性。
~告知客人續住必須免費，因為是 DSRR#DSRR 與免費住宿完全無關。
~強迫客人必須先辦理退房，再重新訂房#這是不必要的負面服務體驗，應有更好的處理方式。
~直接告知客人現場最高的門市房價#雖然可以提供續住價格，但應遵循「不隨意告知現場房價」的謹慎原則，先確認原訂單性質。
}
