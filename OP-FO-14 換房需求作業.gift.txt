$CATEGORY: OP-FO-14 換房需求作業

::OP-FO-14-Q01::當客人因為「房間內有煙味」要求換房時，根據標準作業程序，櫃檯人員首要的處理動作流程為何？ {
~直接告知客房全滿，建議客人使用空氣清淨機。#這忽略了初步評估與展現共感的步驟，且未嘗試解決客人的核心問題。
~請房務部立即前往除味，並請客人在原房間等待。#雖然除味是方法，但規範中提到應先判定狀況並視房況幫客人換房，而非強制客人在原房等待。
=先對造成不便致歉，判定是否有菸味後視房況安排換房。#符合規範中「聆聽並致歉」以及針對煙味問題「判別後視房況換房」的處理邏輯。
~立即要求客人支付清潔費，因為飯店為全館禁菸。#在確認是該客人吸菸前，應先以服務與解決問題為導向，直接收費會造成不佳的服務體驗。
~通知工程部檢查空調系統，判斷煙味是否由管道間傳入。#這是後續修繕的動作，並非第一時間受理需求時對客人的首要作業流程。
}

::OP-FO-14-Q02::若客人因個人喜好（例如：不喜歡房間配置）要求更換至較高階的房型，櫃檯人員應如何處理費用部分？ {
~為了提升滿意度，應直接提供免費升等。#規範中明確指出「客人喜好」造成的換房需報價並收取價差。
~告知客人目前無空房，婉拒其換房要求。#應先查詢系統房況，而非在未查詢前就直接拒絕客人。
~視客人入住天數，決定是否減免該價差。#作業規範中並未提到可依天數隨意減免，標準程序是需報價並收取價差。
=向客人說明作業流程，進行報價並收取房型價差。#針對「客人喜好」導致的換房需求，標準流程要求必須進行報價並收取差額。
~請客人自行在訂房網站上取消並重新預訂。#這會增加客人的操作困難，櫃檯應在 PMS 系統中直接處理換房與帳務。
}

::OP-FO-14-Q03::在進行 PMS 系統轉房作業時，除了確保房費與帳務正確外，哪一項是必須執行的跨部門通知動作？ {
~通知餐廳部，因為客人的早餐券需要重新列印。#雖然可能相關，但並非換房標準作業指導書中規定的必要跨部門通知。
=通知房務部，告知舊房需改為髒房並告知新房號。#這是為了確保舊房能被及時清掃，且房務人員知道客人的新去處，符合流程規範。
~通知安全部，請其派員監控搬遷過程以防財損。#除非有特殊安全疑慮，否則標準搬遷流程不需安全部全程介入。
~通知總經理室，報備每一筆換房記錄以維持品質。#換房屬於日常櫃檯作業，無需逐案呈報總經理室。
~通知資訊部，檢查新房間的 Wi-Fi 連線是否正常。#櫃檯應查詢系統確認是「乾淨房」， Wi-Fi 檢查通常不包含在標準轉房流程的跨部門通知中。
}

::OP-FO-14-Q04::當員工陪同客人進行實際搬遷時，以下哪一組物品是標準流程中明確提醒客人檢查的重點？ {
=保險箱、衣櫃、浴室、冰箱、充電器。#這些是規範中列出的重點檢查區域，也是客人最容易遺留物品的地方。
~電視遙控器、吹風機、瓶裝水、毛巾、室內拖鞋。#這些屬於房間固定備品，並非提醒客人檢查「個人遺留物」的重點。
~窗簾、地毯、燈具、空調開關、電話。#這些屬於房間硬體設備，與搬遷時提醒遺留物的關聯性較低。
~枕頭套、床單、垃圾桶、滅火器、逃生圖。#這些並非客人搬遷時需要特別檢查是否攜帶走的個人物品。
~飯店簡介、原子筆、便條紙、杯墊、攪拌棒。#這些是消耗性文具或耗材，非提醒客人檢查遺留物的核心清單。
}

::OP-FO-14-Q05::若客人表達希望「自行搬運行李」以維護隱私，櫃檯人員應採取何種做法？ {
~堅持由員工搬運，以符合飯店的五星級服務標準。#規範中強調應「尊重客人隱私」，不應強行介入。
=將新房卡交給客人，請其自行搬遷，並約定舊卡歸還時間。#這完全符合規範中對於客人希望自行搬運時的處理建議。
~要求客人在大廳等待，由房務部進入舊房搬運後再通知客人。#這反而侵犯了客人要求的隱私，且非標準作業流程。
~告知客人若自行搬運造成行李損壞，飯店概不負責。#這種語氣不符合服務標準，且會造成客人的負面觀感。
~直接在系統中退房，請客人重新辦理入住手續。#這屬於多餘的程序，只需在 PMS 執行換房操作即可。
}

::OP-FO-14-Q06::換房完成後的「品質追蹤」階段，櫃檯人員應在何時致電新房間詢問滿意度？ {
~搬遷完成後立即撥打，以免客人忘記回饋。#立即撥打可能會打擾正在整理行李的客人，規範建議稍後再撥。
=搬遷完成後 15 至 30 分鐘內。#這是規範中明確設定的關懷回訪時間區間，能確保客人已有時間使用新房設備。
~等到隔天早上客人退房前再詢問即可。#這時效太晚，若新房間仍有問題，將無法及時修補服務失誤。
~搬遷完成後 1 小時以上，給予客人充足的休息時間。#1 小時可能過久，無法在第一時間掌握客人對新房的適應狀況。
~除非客人再次主動聯繫，否則不需撥打關懷電話。#規範要求需主動致電進行關懷回訪，以確保服務品質。
}

::OP-FO-14-Q07::在「結案與品質追蹤」流程中，若換房原因是「飯店設備故障」造成不便，飯店可視情況提供何種補償措施？ {
~提供該次住宿全額免費。#雖然是補償，但規範中建議的是小禮物或延遲退房，全額免費通常不屬於標準慰問範圍。
~強制客人加入飯店會員以獲得點數回饋。#補償應是慰問性質，不應帶有強迫推銷的色彩。
=提供客訴小禮物或安排延遲退房作為慰問。#這是規範中明確提到的服務補償方案，用以修復因故障造成的不便。
~僅口頭道歉，不需提供實質補償，以免養成客人客訴習慣。#若為飯店疏失（設備故障），主動提供慰問是提升滿意度的關鍵步驟。
~更換至總統套房，不論原訂房型等級。#補償應視情況適度調整，過度升等不符合營運成本控制與標準規範。
}

::OP-FO-14-Q08::當客人反映房間設備故障且無法立即排除時，櫃檯人員查詢系統找房時的優先順序為何？ {
~優先尋找最低階的空房以節省成本。#降低等級換房會導致客人不滿，並非優先選項。
~尋找尚未清掃但等級相同的房間。#規範中指明必須尋找「乾淨房」，以確保客人能立即移入。
=尋找同等級或更高一級的乾淨房。#這是規範中確認房況的準則，確保客人權益不縮水甚至獲得小提升。
~請客人到大廳休息，等待原房間修繕完畢。#若無法排解，應以換房為優先，而非讓客人在大廳無止盡等待。
~僅尋找完全相同房號的對稱房間。#房號並不影響換房，重點在於等級是否符合或更高。
}

::OP-FO-14-Q09::在關懷回訪的話術中，除了詢問滿意度，還應特別確認哪一個項目？ {
~確認客人的信用卡額度是否足夠。#這與換房滿意度無關，且會顯得飯店過於急功近利。
~確認客人是否需要預約明日的計程車。#這是額外的服務，並非換房關懷話術中的核心確認內容。
=詢問新房間的設備使用是否都順手。#這是規範中「關懷用語」明確建議詢問的重點，旨在確認新房功能正常。
~詢問客人是否願意在社群媒體上給予好評。#關懷回訪的核心是確保客人舒適，而非在此刻索取評論。
~詢問客人是否需要加購飯店的晚餐行程。#關懷電話應專注於解決先前的問題，不宜進行業務推銷。
}

::OP-FO-14-Q10::若舊房是因為「設備故障」而換房，在結案流程中櫃檯必須同步通知哪一個部門？ {
~財務部，核銷維修費用。#這是後台行政作業，非第一線結案流程中需同步通報的動作。
~業務部，告知可能影響未來的團體訂房。#維修細節不需第一時間通知業務部。
=工程部，進行報修。#規範中明確指出，若為設備故障，應同步通報工程部修繕，以維持房間可用性。
~採購部，申購新的設備零件。#報修應先經由工程部評估，而非由櫃檯直接對接採購部。
~行銷部，追蹤該故障是否影響飯店品牌聲譽。#這過於誇大日常設備報修的通報層級。
}
