$CATEGORY: OP-GS-02 服務禮儀常用語

::OP-GS-02-Q01::當顧客走進館內時，根據「接待問候」的服務標準，下列哪一項做法是正確的？ {
=見到顧客後應在 3 秒內主動問候，並搭配眼神接觸與微笑。#服務標準 1.0 明確規定應在見到顧客 3 秒內主動問候，並強調眼神接觸與微笑的重要性。
~為了表示尊重，應等待顧客先開口詢問後再行回應。#規範中特別強調不可等待顧客先開口，服務人員應主動發起問候。
~若手邊有工作，可先低頭完成後再抬頭向顧客打招呼。#標準作業指導書指出不可低頭不語，這會讓顧客感到被忽視。
~統一使用「您好」即可，不需根據早晚時段區分問候語。#規範要求應依據時段使用「早安」、「午安」或「晚安」等問候語。
~只需點頭示意，不一定要開口說話以維持環境安寧。#作業項目明確要求使用問候語，而非僅以肢體語言示意。
}

::OP-GS-02-Q02::在與顧客確認需求與回應的過程中，下列哪一種話術符合「正向回應」的原則？ {
~「應該可以吧，我幫您看看。」#「應該可以」屬於含糊語句，不符合清楚明確的回應原則。
=「好的，請您稍等，我這邊先為您查詢。」#這句話符合告知稍候並明確說明後續處理動作的標準流程。
~「這部分我不知道耶，要問主管。」#使用「不知道」是禁用語，應改以協助查詢或說明原因的方式呈現。
~「沒辦法喔，現在系統壞掉了。」#直接使用「沒辦法」或否定句過於生硬，應提供替代方案或原因說明。
~「您可以自己看那一張公告，上面寫得很清楚。」#「你自己看」是生硬的命令式用語，在服務禮儀中應被避免。
}

::OP-GS-02-Q03::當發生館方疏失導致客訴時，下列關於「致歉用語」的操作流程，何者最為適切？ {
=先致歉並說明處理方式，且語氣需穩定、不推諉責任。#規範 3.0 指出致歉後必須接續具體處理動作，且態度應積極負責。
~只要誠懇地不斷道歉，顧客通常就能平息怒火。#標準規定致歉後需接續具體處理動作，不可只道歉不處理。
~若非自己部門的錯，應向顧客解釋清楚是由哪個單位負責。#致歉時應保持語氣穩定且不推責，避免將責任推給其他同仁或部門。
~為了不讓顧客久等，應先處理問題，等處理完後再補道歉。#發生異常或疏失時，標準流程是先致歉再說明處理方式。
~可以先使用「不知道耶」來觀察顧客反應，再決定致歉強度。#「不知道」是禁用語，且致歉應在第一時間主動進行。
}

::OP-GS-02-Q04::關於「感謝與送客」的作業標準，下列哪一項行為是嚴格禁止的？ {
=在顧客尚未離開前，即轉身開始做自己的事。#規範 4.0 的注意事項明確指出，不可在顧客尚未離開前轉身處理雜事。
~服務完成後主動向顧客致謝。#這是標準流程中要求的正向服務行為。
~在顧客離開時做出目送的動作。#目送動作是提升服務品質的關鍵禮儀標準之一。
~對顧客說：「謝謝您，祝您今天一切順心。」#這是一句標準且被建議使用的送客語。
~提醒顧客若有其他需要，歡迎隨時告知。#這屬於服務完整性的一環，是符合禮儀的表現。
}

::OP-GS-02-Q05::新進夥伴在教育訓練期間，應優先熟悉並自然使用哪些「七大常用語」？ {
=您好、請、謝謝、不好意思、請稍等、跟您確認、我這邊先為您處理。#這是規範 5.0 中列出的核心服務語句，新進人員需重點訓練。
~您好、請看、等一下、沒關係、不知道、沒辦法、再見。#此選項包含「不知道」、「沒辦法」等禁用語或不恰當用語。
~歡迎光臨、請坐、沒問題、對不起、請自便、你看看、慢走。#此選項語句較為口語，且未完全包含標準要求的七大用語。
~早安、請稍後、謝了、抱歉、確認中、等我一下、幫你弄。#此選項用語過於口語化，不符合服務禮儀的正式標準。
~您好、請教、感謝、抱歉、您確定嗎、我幫您查、待會見。#部分用語與官方規範的「七大常用語」不符。
}

::OP-GS-02-Q06::若顧客提出一個不屬於您負責範疇的問題，根據「禁用語與替代說法」，最適當的回應為何？ {
~「這不是我負責的範圍，請您去問那邊的人。」#直接推責會給顧客留下負面印象，是典型的禁用語。
=「我這邊先幫您確認正確窗口，請您稍等一下。」#這是規範 6.0 建議的替代說法，展現了協助查詢的積極態度。
~「我不清楚耶，您可能要自己去櫃檯問一下。」#同時使用了禁用語「不清楚」與不禮貌的「你自己去問」。
~「這不是我的業務，我也沒辦法幫您。」#「沒辦法」與「不是我負責」皆屬於生硬且不負責任的用語。
~「好的，我幫您處理。」（儘管您不會處理也不找人協助）#這屬於敷衍式回應，雖然開頭正確但後續未落實處理動作。
}

::OP-GS-02-Q07::在執行服務作業時，關於「回應原則」的敘述，下列何者「錯誤」？ {
~應先完整聆聽顧客需求後，再進行重述確認。#這是標準 2.0 規定的正確流程，並非錯誤敘述。
~回應時應保持清楚、簡短、正向。#這是規範中列出的核心回應原則，並非錯誤敘述。
~若需要查詢系統，應先告知顧客稍候再行動。#這是標準流程中要求的必要步驟，並非錯誤敘述。
~即使無法立即完成，也應提供下一步處理方式。#這是規範 6.0 特別提到的處理彈性與責任感，並非錯誤敘述。
=為了展現親和力，應盡量使用過度口語化的在地用語。#規範 5.0 提醒需避免過度口語化的用詞，應以專業的常用語為主。
}

::OP-GS-02-Q08::顧客抱怨櫃檯辦理入住手續等待太久，此時正確的話術應包含： {
~「不好意思，今天客人比較多，所以才讓您久等。」#雖然有致歉，但後續說明帶有推諉給客流量的意味，不符合不推責的原則。
=「不好意思，讓您久等了，我這邊立即為您處理。」#這符合 3.0 先致歉再說明處理動作的標準，展現主動負責的態度。
~「真的很抱歉，我們會再改進，謝謝您的包容。」#僅有致歉與空泛承諾，缺乏「立即處理」的具體行動。
~「請您稍等，我很快就好，不要急。」#語氣帶有命令式（不要急），容易引發顧客反感。
~「不知道前面的人怎麼弄這麼久，不好意思。」#這屬於推諉責任給同仁，是致歉用語中的大忌。
}

::OP-GS-02-Q09::根據規範 6.0，當面臨無法達成顧客要求的情況時，應該如何轉化生硬的「不行」？ {
=改以說明原因、提供替代方案或協助查詢。#這是規範中明確建議將拒絕轉化為專業服務的方式。
~保持沉默並用微笑帶過，避免正面衝突。#沈默不語會讓顧客感到被敷衍，不符合溝通原則。
~反問顧客：「如果是您，您會怎麼處理呢？」#這屬於挑釁式或推卸責任的溝通，極不恰當。
~直接告知這是館方規定，沒有例外。#這種回應過於生硬，未嘗試提供其他協助或說明原因。
~告訴顧客「我沒辦法決定」，請他找更高階的主管。#這屬於「不是我負責」的變形，是缺乏解決問題意願的表現。
}

::OP-GS-02-Q10::服務標準作業中，關於問候時的非語言行為，下列敘述何者正確？ {
~只要聲音宏亮，不一定要看著顧客。#規範要求問候應搭配眼神接觸，以示尊重。
~眼神接觸需持續盯著顧客看，直到對方回應為止。#過度的盯視會造成壓力，適度的眼神接觸搭配微笑才是標準。
=問候時應搭配眼神接觸與微笑，以營造親切感。#這是 1.0 接待問候中明確要求的非語言標準。
~因為戴口罩，所以不需要微笑，只要點頭就好。#即使戴口罩，眼神的笑意與態度依然是服務的重要環節。
~為了效率，可以邊走路邊對顧客喊問候語。#問候應是主動且專注的，匆忙的問候無法體現服務精神。
}
