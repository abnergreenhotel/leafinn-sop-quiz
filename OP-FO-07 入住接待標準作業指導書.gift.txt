$CATEGORY: OP-FO-07 入住接待標準作業指導書

::OP-FO-07-Q01::當顧客表示有預訂但德安系統查無訂單，且確認為 WEB 沒下傳的訂單時，應如何處理？ {
~請顧客聯繫原訂房平台，櫃台不予處理#指導書指出若為 WEB 沒下傳，應嘗試直接下傳或手建新單，而非拒絕處理。
=若系統能下傳就直接下傳，若不能則手建新單#這是處理 WEB 漏單的標準作業流程，以確保顧客能順利辦理入住。
~請顧客取消原訂單並現場重新訂房#這會增加顧客的困擾，並非指導書建議的補救措施。
~立即致電德安系統商要求處理，請顧客在旁等候#第一時間應先解決顧客入住需求，系統問題可後續再行處理。
}

::OP-FO-07-Q02::顧客於下午 $13\:00$ 到達且希望提前入住，若房間已整理完畢，主管判斷需收費，依規定應加收多少費用？ {
~$100$ 元#加休費用的計費標準為每小時 $200$ 元。
~$200$ 元#應從標準入住時間 $15\:00$ 逆推計算至 $13\:00$，共計兩小時。
=$400$ 元#依指導書規定，加休時間為 $15\:00$ 前逆推，一小時 $200$ 元，自 $13\:00$ 起算為兩小時。
~$500$ 元#此金額不符合每小時 $200$ 元的計算標準。
}

::OP-FO-07-Q03::關於大和停車場的補助與操作，下列敘述何者正確？ {
~補助至退房當天的下午 $15\:00$#大和停車場的電腦消磁補助僅到退房當天的中午 $12\:00$。
=進出須輸入相同的四位數字，且需在離場前告知櫃台消磁#這是大和停車場特有的電腦消磁作業流程與顧客須知。
~可以使用折抵券在繳費機掃描出場#使用折抵券的是 TIMES 或模範英才停車場，大和是使用電腦消磁。
~若要多次進出，每次皆由飯店全額補助#原則上每房每晚補助一次，多次短暫進出金額較少時可請客人自行付款。
}

::OP-FO-07-Q04::若特約停車場皆已停滿，引導顧客前往「民權停車場」停放時，下列哪項指示是正確的？ {
~退房後將繳費收據拍照傳給飯店官方 LINE 即可#指導書明確要求需打上統編並回到櫃台請款。
~請顧客自行負擔費用，飯店僅提供地圖資訊#在特約停滿且顧客不再移車的情況下，飯店可提供報帳請款。
=請顧客繳費時務必打上統編，退房時再到櫃台請款#這是針對民權停車場（統聯對面）停滿時的標準補償處理流程。
~告知顧客僅補助路邊停車格，不補助私人收費停車場#若民權停車場也無法使用，才會請客人找路邊或其它停車場。
}

::OP-FO-07-Q05::在辦理入住登記（Check-in）時，針對「外國顧客」必須額外留下哪一項資料？ {
~國外緊急聯絡人電話#指導書並未強調需留下國外緊急聯絡人電話。
~在台聯絡地址#指導書要求的關鍵聯繫資訊是電子郵件。
=E-MAIL#依據入住登記標準，外國顧客須在 R 卡資料中留下 E-MAIL 以供後續聯繫。
~護照影本掃描檔#雖然需查驗證件，但指導書中特別註明需在 R 卡上登記的是 E-MAIL。
}

::OP-FO-07-Q06::若顧客原本有「預先授權」，但入住當天改用 Apple Pay 支付房費，應如何操作？ {
~直接將 Apple Pay 作為預先授權的完成交易#Apple Pay 若非實體卡，一律以銷售交易處理，不可與原授權關聯。
~請顧客聯繫銀行取消，飯店直接刷 Apple Pay 即可#飯店應主動向銀行申請取消預先授權，以釋放顧客的凍結額度。
=以銷售交易刷 Apple Pay，並主動申請取消原預先授權額度#更換支付方式（尤其是非實體卡）時，需取消原授權以免造成重複扣款誤會。
~拒絕更換支付方式，強制顧客使用原授權信用卡#指導書有提供更改付款方式的話術，顯示飯店接受變更支付方式。
}

::OP-FO-07-Q07::在使用德安系統「入訂金」流程中，若顧客續住多日且希望「所有天數開在同一張發票」，發票開立方式應選取？ {
~後開#「後開」通常用於希望一天一張發票的情況。
=已開#依系統操作指南，若客人希望全部開一張發票，應選取「已開」。
~免開#除非特殊稅務狀況，否則住宿皆須開立發票，系統中無「免開」此項說明。
~待開#德安系統作業指導書中定義的選項為「後開」與「已開」。
}

::OP-FO-07-Q08::在德安系統中完成入訂金與備註後，執行最後的 Check-in 步驟應在哪個頁面操作？ {
~訂房管理之訂房管理頁面#訂房管理是用於查詢與建立訂單，實際入住操作需至接待管理。
=接待管理之房間管理頁面的「預計 C/I 清單」#這是執行實體入住（CHECK IN）步驟的標準系統路徑。
~帳務管理之退房結帳頁面#帳務管理與入住流程不同，是在離店時使用。
~房務管理之房間狀況頁面#房務管理頁面主要用於查看清潔狀況，非入住登記的主要操作區。
}

::OP-FO-07-Q09::若住客反應忘記帶房卡且房內「目前無人」，櫃台人員應如何處理？ {
~直接補發一張新卡給住客，請其自行回房#直接給卡有安全風險，應採取更謹慎的處理方式。
~要求住客在櫃台等待其他室友回來#在確認身份無誤後，應協助住客回房而非消極等待。
=核對訂房資訊無誤後，帶領客人上樓進房間#這是處理忘記帶卡且房內無人時的標準安全作業程序。
~請住客出示身分證件影本留存後，方可開啟房門#指導書強調的是核對姓名房號並由人員帶領上樓。
}

::OP-FO-07-Q10::關於入訂金時在德安系統訂單備註的格式，下列何者符合規定？ {
~已收現 $3000$ 元#標準格式需包含「沖訂金編號」。
=沖 $2500001234$，統編#依規範，備註應打上「沖<訂金編號>，統編」，並刪除無用資訊。
~已付清，停車車號 $ABC-1234$#此資訊雖重要，但非系統入訂金流程中規定的備註格式。
~網路訂單已入帳#備註必須連結至德安系統產生的特定訂金編號。
}
