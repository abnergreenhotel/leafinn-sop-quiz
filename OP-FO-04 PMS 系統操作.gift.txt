$CATEGORY: OP-FO-04 PMS 系統操作

::OP-FO-04-Q01::當您正準備接班登入 PMS 系統時，發現無法成功登入，根據標準作業程序，您應該最先採取哪項行動？ {
~直接向資訊部門報修系統故障#直接報修可能忽略了個人帳號設定問題，應先進行初步的自我檢查。
=先行確認帳號密碼與權限設定，確認無誤後再行回報#SOP 規定在無法登入時應先確認帳密與權限，這是排除人為登入錯誤的首要步驟。
~先向同事借用帳號登入，以免耽誤接班進度#規範中明確禁止借用他人帳號登入，這涉及資安與操作責任歸屬問題。
~嘗試重新啟動電腦並多次嘗試輸入不同密碼#多次錯誤嘗試可能導致帳號鎖定，且未依程序確認基本設定。
~跳過登入步驟，先以紙本方式記錄當班作業#這不符合系統操作的異常處理流程，應優先解決登入問題或依規範回報。
}

::OP-FO-04-Q02::一位顧客致電詢問其訂房內容，但您輸入姓名後卻查不到訂單。此時，您應如何調整搜尋條件以進一步確認？ {
~重新整理網頁畫面，再次以相同姓名查詢#若條件不變，單純重新整理通常無法解決查無資料的問題。
~請顧客提供身分證字號，這是系統唯一的查詢依據#資料中提到的查詢條件包含姓名、編號、日期、房號，並未將身分證字號列為唯一依據。
~直接告訴顧客可能未訂房成功，請其聯繫訂房平台#在未窮盡查詢方式前直接拒絕顧客，可能造成服務糾紛且不符合操作指引。
=將訂房卡狀態由「有效訂房」更改為「全部」再進行搜尋#SOP 指出，若找不到訂單，切換至「全部」狀態可查看到除有效訂房以外的資料。
~詢問顧客是否在其他分館訂房，並切換至其他館別查詢#雖然是一個可能原因，但應先在原系統內窮盡所有狀態搜尋條件。
}

::OP-FO-04-Q03::在與顧客溝通進行訂單查詢時，下列哪一種對話話術最符合服務標準作業指導書？ {
~「請問您是什麼時候訂房的？我找找看。」#語氣較不正式且未引導顧客提供最具效率的查詢關鍵資訊。
=「您好，請問您的訂房大名或訂單編號是什麼呢？我這邊幫您查詢。」#這是 SOP 規範的標準話術，同時引導顧客提供關鍵搜尋資訊。
~「小姐/先生您好，您的房號是多少？我幫您看帳款。」#在未核對身份前直接切入房號與帳務資訊，不符合資訊核對的注意事項。
~「請稍等，系統有點慢，我正在幫您查。」#話術應聚焦於獲取正確資訊，而非向顧客抱怨系統效能。
~「不好意思，您可以直接唸訂單編號給我嗎？這樣比較快。」#雖然有效率，但缺乏標準的問候語與多元的查詢選項引導。
}

::OP-FO-04-Q04::關於 PMS 系統中的「備註與資料更新」，下列操作何者正確？ {
~為了節省空間，應使用只有自己看得懂的代碼進行備註#備註需簡潔明確，不可使用模糊字眼，以免造成其他夥伴誤解。
=若修改重要資訊，應於備註開頭加上修改日期#SOP 明確規定修改重要資訊時需加上日期，例如「5/1客人致電...」，以便追蹤。
~備註欄位可以隨意填寫，不需經過再次核對#修改前應再次核對顧客資訊與更新內容，確保資料準確性。
~所有顧客需求，無論大小，皆只需記錄在個人筆記本中#應依實際需求在 PMS 系統內新增或更新備註，以利團隊資訊共享。
~修改訂單時，應先刪除舊有備註以保持介面整潔#任意刪除歷史備註可能導致後續爭議時缺乏追溯依據。
}

::OP-FO-04-Q05::當您在使用系統時發現「串單異常」或「畫面錯誤」，而主管暫時不在場，您應該如何處理？ {
~嘗試點擊所有功能按鍵，看看是否能恢復正常#系統異常時不可自行亂試，以免造成資料庫錯誤或更嚴重的損壞。
=先行截取關鍵資訊畫面並回報，依 SOP 判斷是否轉手建單#這是異常處理的標準步驟，旨在保留證據並尋求正確的替代方案。
~直接請工程師進入後台修改，不需經過內部回報#應先截取資訊並依內部 SOP 處理或向主管回報，而非直接跨權限操作。
~關閉系統，等待下一班夥伴來處理#消極避讓會影響現場營運，不符合異常處理的積極原則。
~告訴顧客目前系統故障無法提供任何服務#應採取「先確認再回覆」的話術，而非直接拒絕服務。
}

::OP-FO-04-Q06::顧客致電確認訂房，但您當下在系統中找不到訂單且通話已進行一段時間，下列做法最專業且符合流程的是： {
~請顧客保持通話不要掛斷，直到您找到為止#這會讓顧客在電話中枯等，降低服務滿意度。
~直接告知顧客查無資料，請其稍後再撥#這種處理方式太過被動，且可能導致顧客流失。
=先留下顧客電話，告知後續處理完畢後會再次致電回覆#SOP 建議若當下找不到訂單，應先留電話後處理，避免顧客在線上空等。
~隨便找一張訂單先應付顧客，等掛斷後再慢慢找#這屬於嚴重的資料核對錯誤，違反不可憑模糊資訊揭露內容的規定。
~要求顧客將訂房確認信傳真過來，否則無法處理#應先嘗試內部系統查詢與積極聯繫，而非一開始就將責任推給顧客。
}

::OP-FO-04-Q07::關於「基本介面辨識」的作業指導，下列哪項行為是受禁止的？ {
=在尚未熟悉欄位功能前，任意修改正式資料#注意事項明確規定，未熟悉欄位前不可任意修改，以防資料錯誤。
~反覆切換房況畫面與住客資料以求熟悉#熟悉畫面切換是學習基本操作的一部分，並非禁止事項。
~查詢帳務欄位時核對顧客的姓名#查詢時核對資訊是正確的服務準則。
~觀察資深夥伴如何操作常用功能入口#向資深夥伴學習是 SOP 鼓勵的行為（如遇不確定操作時）。
~在測試環境中進行各種功能的演練#正式值勤前完成演練是必要的，不屬於禁止行為。
}

::OP-FO-04-Q08::在「常用功能操作」中，若遇到不確定的操作步驟，應如何處理？ {
~查閱個人筆記，若無記載則憑感覺操作#憑感覺操作容易導致系統資料錯誤，並非正確流程。
=先詢問資深夥伴或主管#SOP 指出，遇不確定操作時，應先尋求專業夥伴或主管的協助。
~直接聯繫軟體供應商請求遠端支援#這通常是最後的技術手段，應先進行內部求援。
~放棄該項操作，等下一次有機會再試#應及時解決不確定性，以利業務順利進行。
~參考網路上其他 PMS 系統的操作影片#不同系統操作方式各異，應以本館 SOP 與內部指導為主。
}

::OP-FO-04-Q09::進行「訂單查詢」時，下列哪一項資訊是不應僅憑模糊資訊就直接透露給對方的？ {
~飯店的地址與聯絡電話#飯店公共資訊不屬於需要嚴格核對才能透露的訂單內容。
~當天的剩餘空房數量#雖然重要，但與「訂單內容」的核對保護性質不同。
=顧客的付款狀態、備註或個人詳細訂單內容#注意事項規範，不可憑模糊資訊直接透露訂單內容，須確保資訊安全。
~系統目前的登入館別#這是內部操作狀態，並非需要對外保護的客資內容。
~PMS 系統的版本號碼#版本號碼不涉及顧客隱私，並非禁止透露的核心資訊。
}

::OP-FO-04-Q10::在執行「備註與資料更新」時，當顧客提出停車需求與預計抵達時間，您的備註應符合什麼原則？ {
~越詳盡越好，包含顧客說話的語氣也要記錄#備註應保持簡潔明確，過多贅詞反而會干擾重要資訊的讀取。
~僅需記錄在自己的值班日誌中，不需更新至系統#系統備註是為了讓全體夥伴皆能掌握即時需求，必須登載於 PMS。
~先幫顧客預設好所有服務，等顧客來再確認#應依「實際需求」新增，且修改前需再次核對內容。
=簡潔明確且不可使用模糊字眼，並於修改前再次核對#這是 SOP 中關於備註與資料更新的核心要求，確保資訊的可讀性與準確性。
~儘量使用英文備註，以顯專業度#文件未要求語言，但重點在於「簡潔明確」與「不可模糊」，語言非首要考量。
}
