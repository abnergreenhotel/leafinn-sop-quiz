$CATEGORY: OP-FO-28 寄售商品作業

::OP-FO-28-Q01::在處理「展覽合作」商品的銷售時，下列關於付款方式的敘述何者正確？ {
=僅限使用現金支付#根據流程規範，展覽合作商品僅能接受現金付款，後續再由櫃台補入德安系統。
~可選擇信用卡或現金支付#作業規範中特別強調此類商品客方僅能使用現金，不可使用信用卡或其他電子支付。
~統一由旅客退房時掛帳處理#文件註記展覽合作商品需收現並放入當班帳包，並非一律掛帳處理。
~僅接受行動支付以利系統對帳#作業指南明確指出只能使用現金，並非行動支付。
~視合作商要求決定支付方式#SOP 已統一規定為現金支付，而非彈性視情況決定。
}

::OP-FO-28-Q02::進行「扣商品庫存」作業時，第一步操作程序為何？ {
=開啟雲端資料夾#扣庫存的標準作業流程起始於開啟雲端資料夾以進行後續資料登錄。
~直接在德安系統修改數量#德安系統主要用於入帳，庫存扣除需先透過雲端資料夾進行記錄。
~致電主管核准扣額#扣庫存為一般銷售程序，除非商品破損，否則不需每筆都致電主管。
~填寫紙本領用清單#作業流程中使用的是雲端資料夾，而非傳統紙本清單。
~掃描商品條碼#根據手冊指示，庫存管理是透過搜尋品項名稱後手動輸入購買數量。
}

::OP-FO-28-Q03::若商品發生損壞導致無法販售，櫃台人員應如何應對？ {
=請客人直接照價賠償，並視情況請主管處理#流程規定當商品損壞時應請客人按原價賠償，且必要時須介入管理職權。
~自行吸收損失，將商品直接報廢#手冊中明確要求需向客人提出照價賠償，並非直接報廢處理。
~將損壞商品退回給原廠商#損壞賠償應由現場責任歸屬方（如客人）負擔，而非直接退回廠商。
~以五折價格促銷售出#文件指出損壞商品已「無法販售」，故不應以促銷方式處理。
~請客人自行聯繫廠商維修#飯店應直接與客人對接賠償事宜，而非讓客人聯繫廠商。
}

::OP-FO-28-Q04::在德安飯店前台系統中進行入帳時，進入「旅客帳處理」前的正確路徑為何？ {
=接待管理 ⮕ 房間管理 ⮕ 找到該名旅客後按右鍵#這是德安系統中定位特定旅客並進行帳務操作的標準導航步驟。
~財務管理 ⮕ 應收帳款 ⮕ 旅客查詢#作業指南指定路徑為接待管理下的房間管理，而非財務管理模組。
~系統設定 ⮕ 參數維護 ⮕ 帳務設定#入帳屬於日常接待流程，不涉及系統層級的參數維護。
~櫃台結帳 ⮕ 日結作業 ⮕ 消費錄入#日結作業是收班時的操作，銷售入帳需在房間管理中即時處理。
~訂房管理 ⮕ 訂單查詢 ⮕ 旅客名單#雖然訂房管理有旅客名單，但手冊指定的操作入口是房間管理。
}

::OP-FO-28-Q05::根據作業指導書，關於「電子發票列印」的特殊規定為何？ {
=只有二館需要進行此步驟#流程中特別標註只有二館在結帳完成後需要點選列印電子發票。
~所有分館皆必須列印紙本發票#指導書中明確標註「只有二館要」，表示這並非所有分館的共通程序。
~僅限於購買展覽商品時列印#列印發票的規範是與館別掛鉤，而非與商品類別掛鉤。
~若客人不索取則不需點選列印#程序上要求特定館別須點選確定，並非視客人意願決定。
~每月結帳時統一補開#發票是在銷售入帳、結帳流程的最後一步即時處理的。
}

::OP-FO-28-Q06::當客人購買展覽合作商品後，收取的現金應如何保管？ {
=放在當班帳包內即可#作業標準是將展覽商品收到的現金存放在當班的帳包中，以利後續對帳。
~直接存入銀行自動存款機#飯店現金流管理需先入帳包統計，而非由個別櫃台人員直接存款。
~暫存於櫃台抽屜，下班交給警衛#帳務現金應放入專用的帳包內，而非隨意存放在抽屜或移交給非會計/主管人員。
~交給原展覽合作商帶回#現金應先保留在館內，待月結後再由會計與廠商進行結算。
~換成零錢存入櫃台零用金#這是銷售收入而非零用金撥補，應獨立放入帳包保管。
}

::OP-FO-28-Q07::月結作業的主要流程中，主管的角色負責下列哪一項任務？ {
=與廠商盤點核對並結算銷售金額#主管需每月定期與廠商對帳並結算，完成後再將資料提供給會計。
~負責將每筆交易登入德安系統#系統入帳通常由銷售當下的櫃台人員負責，而非主管在月結時補做。
~代表飯店支付現金給廠商#主管負責對帳與結算，最終的款項撥付通常由會計或財務部門執行。
~向客人追討破損賠償費用#這是發生損壞時的即時處理，並非月結作業的常態內容。
~親自補齊所有客房備品庫存#備品補齊是日常營運事項，月結重點在於與外部廠商的帳務核對。
}

::OP-FO-28-Q08::在德安系統中，「入帳」之後至「結帳」之前的關鍵動作為何？ {
=選擇付款方式為現金#在入帳程序完成後，必須點選結帳並選擇「現金」作為付款方式。
~重新確認雲端資料夾數量#庫存扣除是在入帳前完成的步驟，結帳時應專注於系統內的帳務操作。
~列印銷售報表#報表通常在日結或月結才產生，單筆結帳時不需要此動作。
~請主管刷授權卡#標準流程中並未提到一般商品銷售結帳需要主管授權卡。
~將商品包裝並交給客人#這是實體交付動作，此題考點在於德安系統內的操作邏輯步驟。
}

::OP-FO-28-Q09::為何在德安系統入帳的「備註」欄位需寫上賣出商品名稱？ {
=以利主管進行對帳核對#在備註欄標記商品名稱是為了讓主管在後續查帳時能明確對應銷售品項。
~方便會計開立收據給客人#發票/收據是根據系統消費項生成的，備註主要是用於內部核對。
~作為廠商的保固憑證#備註內容主要對應內部對帳，而非提供給客人的保固文件。
~紀錄客人的購物偏好#這是帳務處理流程，主要目的是財務核銷，而非顧客關係管理(CRM)。
~避免重複計價#避免重計是透過系統數量控制，備註是為了審核方便。
}

::OP-FO-28-Q10::關於庫存管理，作業流程中規定應多久執行一次「定期補齊商品庫存」？ {
=未明訂天數，但需定期執行#手冊中僅註明「需定期補齊商品庫存」，並未硬性規定每日或每週。
~每小時檢查一次#過於頻繁且手冊未如此要求，只需確保庫存充足即可。
~只有在主管交辦時才執行#這是櫃台的常態性注意項目，不應被動等待指示。
~每個月結帳後才補一次#庫存若等到月結才補，可能會造成平日銷售的中斷。
~當客人詢問時才去確認#應主動定期補齊，以確保現場隨時有貨可售。
}
