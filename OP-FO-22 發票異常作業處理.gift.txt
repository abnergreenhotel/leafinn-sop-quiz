$CATEGORY: OP-FO-22 發票異常作業處理

::OP-FO-22-Q01::在進行「檢查發票跳開」作業時，若發現發票號碼與日期不符（跳開）的情形，應優先如何處理？ {
~直接手寫註記修正跳開的發票號碼#發票異常不應以手寫修正，而需遵循系統化的異常處理規範。
=依照作廢流程作廢發票，並由後台重新開立#作業規範明確要求，若檢查發現跳開發票，需先執行作廢流程，再參照後台開立流程重新開立。
~將該發票留至月底統一報廢處理#異常發票應即時發現即時處理，避免影響後續報表核對與帳務正確性。
~僅在備註欄位輸入跳開原因即可，無需作廢#單純備註無法解決發票號碼不連貫或日期錯亂的異常狀態。
~手開一張同樣金額的三聯式發票覆蓋舊發票#手開三聯發票有其特定使用情境，不應用於補救系統跳開的異常。
}

::OP-FO-22-Q02::執行「作廢發票流程」時，在德安系統中點選作廢發票後，下一個關鍵的查核步驟是什麼？ {
~直接將發票撕毀並丟棄#作廢發票必須保留並貼入表格，不可隨意丟棄。
~通知主管並請主管核對金額#這是管理程序，而非 SOP 中規範的系統操作查核步驟。
=離開後重複以上動作複查是否作廢成功#為了確保系統狀態已更新，作業規範要求完成作廢後須再次進入系統確認狀態。
~立即點選報表列印當日作廢清單#列印清單通常是結帳時的動作，並非作廢單張發票時的立即複查步驟。
~手動修改該張發票的開立金額為 0#作廢功能與修改金額是不同操作，正確程序是直接點選右鍵作廢發票。
}

::OP-FO-22-Q03::關於「後台開立發票」的設定，在銷售點代號的選取上，應選擇哪一個選項？ {
~總務部櫃台#總務部通常不負責前檯或客房發票的直接開立。
~餐飲部櫃台#本作業規範是針對客房部的發票異常作業處理。
~一館櫃台#雖然也是櫃台，但 SOP 圖示中特別標註的是二館特定的代稱。
=客房櫃檯（二館櫃台）#根據 SOP 執行說明第 5 點，銷售點代號應正確選取「客房櫃檯（二館櫃台）」。
~財務部後台#後台開立是功能名稱，但代號仍需選取對應的銷售營業點。
}

::OP-FO-22-Q04::若顧客入住天數為三天兩夜，根據作業注意事項，發票開立的標準數量為何？ {
~必須每天開立一張，共三張#每日開一張會造成行政負擔，且不符合 SOP 的簡化原則。
=顧客入住一天以上，發票只要一張#作業規範中特別註記（*1-1），顧客入住一天以上，發票原則上合併開立一張即可。
~第一天開一張，退房當天再補一張#分兩次開立並非標準流程要求的做法。
~依據顧客退房時的要求，若沒要求則不開#無論顧客是否要求，均需依法開立發票。
~由顧客決定要開一張還是按天數分開開立#雖然可彈性處理，但標準 SOP 是以一張為原則。
}

::OP-FO-22-Q05::在「手開三聯流程」中，若房價總計為 $2100$ 元，請問「銷售額」應該如何計算？ {
~直接填寫 $2100$ 元#總計金額包含了營業稅，銷售額需將稅額剔除。
~以 $2100$ 乘於 $0.05$ 作為銷售額#乘於 $0.05$ 是計算稅額的方法，而非銷售額的推算。
=以 $2100$ 除以 $1.05$ 等於銷售額#根據計算方式規範，銷售額 \= 房價 $\\div 1.05$。
~以 $2100$ 減去 $100$ 元計算#扣除固定金額並非法律規定的營業稅內含計算公式。
~以房價乘以 $0.95$ 作為銷售額#這是常見的折扣計算誤區，而非正確的稅前銷售額算法。
}

::OP-FO-22-Q06::完成手開三聯發票後，應將哪幾聯交給顧客？ {
~第一、二、三聯全部給顧客#第一聯為存根聯，必須由開立單位保留。
~僅交付第三聯（收執聯）#三聯式發票若為報帳使用，通常需要交付扣抵聯與收執聯。
=交付第二、三聯給顧客#規範中明確指出，手開三聯發票完成後，第二聯（扣抵聯）與第三聯（收執聯）應交給顧客。
~僅交付第一聯（存根聯）#存根聯應留在發票本中備查，不應交予顧客。
~顧客僅需在發票本上簽名，不需拿走任何聯式#發票是交易憑證，必須將法定聯次交給買受人。
}

::OP-FO-22-Q07::關於作廢發票的實體處理，下列哪項敘述完全符合規範？ {
~蓋上作廢章並妥善收納於抽屜#僅收納於抽屜是不夠的，還需進行黏貼管理。
=蓋上作廢章並黏貼到作廢發票表格#SOP 要求已作廢發票需蓋上作廢章，並完成黏貼於表格的動作以便後續核銷。
~無需蓋章，只要在系統作廢即可#實體發票必須有作廢標記，否則有誤用或漏記的風險。
~將發票對折後直接丟入回收桶#作廢發票是重要憑證，必須保留實體核銷，不可丟棄。
~將發票黏貼於當日的日結報表後方#規範要求黏貼於特定的「作廢發票表格」，而非日結報表。
}

::OP-FO-22-Q08::在執行「檢查發票跳開」作業時，應該選取哪一段日期區間進行查詢？ {
~當週週一至今日#區間過長會降低每日稽核的效率，通常只需檢查近期。
=昨日及今日#SOP 規定選取發票日期（昨日及今日）查詢，以核對發票號碼與日期是否跳開。
~今日整點至目前時間#僅查今日可能漏掉跨日（夜班）產生的跳開問題。
~本月 1 日至今日#這是月底核帳範圍，非每日例行檢查發票跳開的區間。
~上週同一天與今日#這樣的日期組合不具連續性，無法有效核對號碼跳開。
}

::OP-FO-22-Q09::在「手開三聯流程」中，除了填寫基本資料與計算稅額外，哪一項行政細節是絕對不能遺漏的？ {
~必須使用紅筆填寫#一般填寫發票並無規定必須使用紅筆，通常以黑色或藍色原子筆為主。
~蓋上飯店的店章#規範中特別強調的是針對買受人資訊的章戳。
=蓋上統編章#依照 SOP 第 7 點，必須蓋上統編章以符合憑證開立規範。
~註明顧客的聯絡電話#聯絡電話並非發票上的法定必填欄位。
~在發票背面簽名確認#發票簽名通常出現在領收證明，而非三聯式發票開立的標準動作。
}

::OP-FO-22-Q10::開立三聯式發票時，注意事項中提到的「日期是否回開」主要是在提醒夥伴什麼？ {
~注意是否超過發票開立的營業稅期#雖然稅期重要，但「回開」是指發票號碼與日期的前後邏輯一致性。
~檢查發票號碼是否有漏開的情形#漏開與回開是不同概念，回開指的是日期倒置的問題。
~注意日期是否填寫成明年的日期#這是筆誤問題，而非 SOP 強調的「回開」核對重點。
=核對日期是否早於前一張發票，造成序號與日期不符#回開發票會導致發票開立的時間軸與號碼順序衝突，是稅務查核的重點。
~確保發票日期是顧客預約的日期#發票應以實際結帳或開立日為準，而非預約日。
}
