$CATEGORY: OP-FO-19 OTA 平台管理

::OP-FO-19-Q01::情境：今日適逢當地舉辦大型音樂節，館內住房率預計極高。根據作業標準，針對「平台房量與價格巡檢」應如何處理？ {
~維持每日一次的例行巡檢即可，避免過度干擾系統。#作業規範明確指出，在高住房日應提高巡檢頻率，而非維持例行頻率。
=應提高巡檢頻率，確保主要 OTA 平台與 PMS 庫存一致。#針對重要節日或高住房日，作業流程要求增加巡檢次數，以防房況與售價產生落差。
~僅需巡檢主要 OTA 平台，不需與內部 PMS 系統進行比對。#流程 1.0 的執行說明強調必須比對 PMS 與 OTA 之庫存是否一致。
~關閉所有 OTA 平台，統一由現場訂房，以省去巡檢時間。#流程中並未建議僅因節日就關閉所有平台，而是應加強管理與預防超賣。
~將巡檢工作全權交由主管執行，第一線人員僅需處理訂單。#巡檢屬於 OTA 平台管理的基本作業項目，應由負責夥伴依規範執行。
}

::OP-FO-19-Q02::在執行「浮動房價調整」作業時，下列哪一項是調整前**不一定要**確認的項目？ {
~適用的房型範圍。#根據流程 2.0，調整前確認房型是標準步驟之一。
~調整價格的適用日期。#日期是價格調整最核心的要素，必須在操作前確認。
=當日值班房務人員名單。#房價調整主要參考住房率、節日與競品狀況，與當班房務名單無直接關聯。
~適用的平台範圍。#不同平台的客群與合約可能不同，調整前需確認適用的平台範圍。
~主管的調整指示。#流程規範中提到調整需依據主管指示，以確保調價符合策略。
}

::OP-FO-19-Q03::關於「超賣預防」的操作，若發現某特定日期房況已趨於緊張，下列哪項做法符合規範？ {
~立即刪除所有未付款的預約，以釋出空房。#擅自刪除預約可能違反平台規則並損害顧客權益。
~調高該日售價至最高價，讓客人自然不願意訂購。#雖然可間接降低訂購率，但正式的預防措施應包含調整售數或關房。
~直接停止巡檢，直到房況恢復穩定。#房況緊張時反而應增加關注頻率，而非停止。
=依指示調整停售、關房或下修可售數。#這是流程 3.0 中明確規定的預防性操作，可有效控制超賣風險。
~在平台留言板註明「已無空房」，但不修改系統售數。#若不修改售數，系統仍會允許訂單流入，無法達到預防效果。
}

::OP-FO-19-Q04::一旦確認發生「超賣（Overbooking）」，夥伴的第一個處置步驟應為何？ {
~自行聯繫顧客要求取消訂單，避免留下負評。#超賣案件需依指示由主管介入，不可擅自要求顧客取消。
~先處理當日其他客人的 Check-in，等有空再處理超賣。#作業規範強調發現可疑超賣時不可拖延，需立即處理。
=立即整理受影響訂單資訊（來源、房型、付款狀態）並回報主管。#流程 4.0 規定確認超賣後應立即整理完整資訊並回報主管，以便採取應變措施。
~直接將顧客轉至其他飯店，再事後告知主管。#轉單處理需在主管指示下進行，並應先回報情況。
~在 PMS 系統標註異常後，等待隔日交班處理。#超賣具有時效性，必須立即處理，不可延宕至隔日。
}

::OP-FO-19-Q05::在「超賣應變處理」中，對受影響顧客進行說明時，最重要的態度原則是什麼？ {
~告知這是平台系統的錯誤，請顧客向平台申訴。#流程強調不可讓顧客感受推託，將責任完全推給平台屬於推託行為。
=語氣穩定，展現積極解決的態度，不可讓顧客感受推託。#根據注意事項 4.0，穩定的語氣與不推託的態度是處理危機的關鍵。
~為了讓顧客諒解，可以適度誇大受損情況。#溝通應誠實穩定，誇大事實可能導致後續爭議。
~保持強硬立場，強調飯店保留隨時變更訂單的權利。#強硬立場會激化矛盾，不利於超賣後的協調與補救。
~儘量縮短通話時間，不讓顧客有提問的機會。#這屬於消極處理，會損害品牌形象並加深顧客不滿。
}

::OP-FO-19-Q06::關於「平台訊息與訂單追蹤」，若遇到需要聯繫顧客的情況，應優先採取何種方式？ {
~肉搜顧客的社交媒體直接留言聯繫。#此舉涉及隱私侵犯，非標準作業流程。
~直接撥打顧客手機號碼，無論時間早晚。#應先參考平台規則進行聯繫，避免不必要的隱私與騷擾爭議。
~忽略該需求，等待顧客主動來電詢問。#流程 5.0 要求留意後台訊息與客服通知，主動跟進是職責。
=依平台規則留言或轉由平台客服協助聯繫。#這是流程 5.0 規定的標準做法，能確保通訊紀錄留存於平台後台。
~私下加入顧客的通訊軟體（如 LINE）進行協調。#未透過平台或官方管道聯繫可能導致後續糾紛時缺乏官方紀錄佐證。
}

::OP-FO-19-Q07::在下班前進行「異常紀錄與結案」時，若發現有一個平台異常案件尚未結案，應如何處理？ {
~將案件留在交班表，不需特別口頭說明。#流程 6.0 的注意事項強調，交班時必須清楚說明未結案件。
~為了準時下班，直接在系統中標註「已結案」。#未結案案件若標註已結案，會導致後續無人追蹤，造成更大損失。
=將該案件紀錄於交班表，並確認已有接手人且交代清楚細節。#流程 6.0 規定需紀錄於交班或追蹤表，並確認有接手人且交代清楚，才算符合程序。
~帶回家利用休假時間繼續聯繫，不需交接。#公務應於館內完成交接，避免私人時間處理導致資訊斷層。
~直接通知主管案件失敗，申請結案。#在未盡全力協調或交接前，不應隨意放棄處理。
}

::OP-FO-19-Q08::根據流程 2.0 的「注意事項」，調整浮動房價時最核心的紀律為何？ {
~只要覺得市場需求強勁，即可隨時調漲房價。#雖然參考市場，但作業指導書強調調價需有依據與紀錄。
~房價調整僅需更改 PMS 系統，OTA 會自動同步。#調整後必須再次檢查平台是否正確生效，不能單方面依賴系統自動化。
=調價需有依據與紀錄，不可任意變動。#這是流程 2.0 注意事項中明確要求的規範，目的是為了後續追蹤與決策分析。
~為了競爭力，應隨時設定比對手低 $10$ 元的價格。#房價調整需依據多方因素與主管指示，不應採取盲目的單一價格戰策略。
~調整後若發生錯誤，可宣稱為系統異常而不需負責。#規範要求調整後需再次檢查，以確保操作的正確性。
}

::OP-FO-19-Q09::在「超賣預防」中，除了高住房日外，還有哪些日期需要「提前檢查」是否有超賣風險？ {
~只有連假和過年才需要檢查。#除了節日，團體進房日與多平台同步販售日也屬於高風險期。
=團體進房日與多平台同步販售日期。#根據流程 3.0，團體進房與多平台販售會大幅增加庫存扣減的速度與複雜度，需提前預防。
~全館空房超過一半的平日。#空房充裕時，超賣風險相對極低。
~館內設備維護日。#維護日主要關心維修流程，而非庫存與超賣預防。
~每週一固定檢查即可，不論是否有活動。#預防作業應針對高風險日期提前執行，而非僅採固定週期。
}

::OP-FO-19-Q10::對於「平台訊息與訂單追蹤」，當收到關於「無效信用卡」的通知時，根據標準作業流程應如何處理？ {
~立即在 PMS 系統中手動取消該訂單。#必須先依平台規則處理或通知顧客，不可直接取消。
~不予理會，等待顧客 Check-in 時再刷卡。#若為無效卡，會造成保證金扣款失敗，屬於需要追蹤處理的異常訊息。
=將其視為一般待處理訊息，於時限內處理。#無效信用卡屬於流程 5.0 中需留意並於規定時限內處理的平台訊息。
~直接打電話給顧客，請其提供口頭信用卡卡號。#基於資安原則與平台規則，通常不建議透過電話索取完整卡號。
~在平台留言回覆「刷卡失敗」，然後結案。#留言後需持續追蹤直至問題解決（換卡或取消），而非直接結案。
}
