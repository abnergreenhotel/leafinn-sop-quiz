$CATEGORY: OP-HK-12 設備損壞回報

::OP-HK-12-Q01::在客房內發現牆面插座無法供電，準備進行照片紀錄時，除了拍下損壞設備本身，還必須在照片中完成哪項操作？ {
=使用紅色標記損壞的具體位置#根據照片紀錄作業規範，對於如插座無反應等靜態設備問題，必須以紅色標記損壞處以利後續辨識。
~將房務工作卡放在旁邊對比位置#雖然放置參照物可能有助於定位，但作業規範明確要求的是使用紅色標記，而非工作卡。
~連續拍攝三張不同距離的照片#規範中並未要求拍攝特定數量的照片，重點在於必須有損壞位置的紅色標記。
~在照片下方手寫註記損壞時間#時間資訊通常包含在訊息回報或系統中，規範要求照片本身的處理應著重於位置標記。
}

::OP-HK-12-Q02::房務員在公區巡檢時發現消防指示燈出現閃爍情形，根據流程應如何記錄此異常？ {
=拍攝一段短影片紀錄動態狀況#對於燈光閃爍、異音或漏水等動態問題，短影片能更準確地呈現問題現狀。
~拍攝三張不同角度的高清照片#靜態照片難以呈現燈光「閃爍」的具體頻率與狀況，因此影片紀錄較為適當。
~直接通知水電技工到場查看#直接通知技工前應先依據流程完成影像紀錄與群組回報，以留存設備異常證據。
~在群組內用文字詳細描述閃爍頻率#文字描述可能存在主觀誤差，作業規範規定此類問題應透過短影片方式紀錄。
}

::OP-HK-12-Q03::在進行損壞設備的「群組回報」時，下列哪一項是必須包含在訊息中的核心內容？ {
=是否已進行初步處理測試#回報內容必須包含：位置、損壞原因、是否做過初步測試以及是否需要主管協助，以利判斷後續處理優先度。
~損壞設備的原始購入價格#採購資訊並非現場回報流程所需的即時資訊，重點應放在現況與初步處理結果。
~預計修復所需的作業時間#修復時間通常由專業維修人員判斷，房務回報的重點在於精準傳達受損現況。
~該房間前一位住客的姓名#雖然損壞可能與住客有關，但 SOP 規定的回報核心內容不包含住客個人資料。
}

::OP-HK-12-Q04::若遇到週六等連續假期「趕房」期間發現設備損壞，為了確保作業效率，回報方式應如何調整？ {
=即時回報損壞原因並請主管立即協助#假日趕房時可簡化流程，透過即時回報原因讓主管介入協助，以確保客房能準時交付。
~等當天所有房間整理完後統一回報#延後回報會導致維修延遲，影響客房供應，應在發現時即時請求主管支援。
~略過影像紀錄，僅口頭告知下一班#口頭告知容易遺漏資訊，假日趕房雖可求快，但仍應即時向主管回報損壞原因。
~先自行嘗試拆解修復以節省時間#房務員應專注於趕房與初步測試，擅自拆解設備可能造成二次損壞或安全問題。
}

::OP-HK-12-Q05::當房務員發現設備損壞（如電視無法開啟）時，執行的第一個動作應該是什麼？ {
=確認損壞位置原因並做初步測試#作業流程的第一步是確認損壞具體狀況，並透過初步處理測試排除簡單的操作問題。
~立即拍影片上傳至房務群組#影像紀錄應在初步確認與測試原因之後才進行，以確保影像內容能準確反映問題。
~搬動另一台電視進入房間替換#在未確認損壞原因與獲得指示前，不應擅自搬動大型設備，應先執行初步測試。
~聯絡工務單位直接進行維修#應先依照 HK (房務) 內部 SOP 流程進行初步確認與回報，再由主管協調專業維修。
}
