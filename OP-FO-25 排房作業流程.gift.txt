$CATEGORY: OP-FO-25 排房作業流程

::OP-FO-25-Q01::在進行排房作業時，若遇到同一姓名預定多筆訂單的情況，下列哪一項是標準的操作原則？ {
=應優先進行排房，並盡量安排在同一樓層#根據作業指導書，多筆同名訂單應先行排房，且為了便利性及管理，應盡量集中在同一樓層。
~需等候所有單筆訂房排定後，再處理多筆訂單#流程中明確指出多筆訂單應「先行排房」，而非最後處理。
~應分散排在不同樓層，以降低音量影響#流程建議多間房間者盡量排在同樓層，分散樓層反而不便於同行住客往來。
~必須排在高樓層以滿足所有多筆訂單顧客#高樓層屬於「特殊需求」，應視訂單備註而定，並非所有多筆訂單都強制排在高樓層。
~直接交由早班人員於當日確認即可，不需預先安排#排房流程包含入住前流程，應預先作業，早班僅負責最後的再確認。
}

::OP-FO-25-Q02::列印「旅客登記卡 (R卡)」時，若遇到同批顧客有多間房號，下列關於勾選作業的敘述何者正確？ {
=點選全選後，將數字最大的房號取消勾選後列印#作業程序規定為避免重複或特定行政作業需求，需取消勾選同批顧客中數字最大的房號。
~點選全選後，直接按列印以確保每間房都有R卡#這會導致多筆同名訂單重複列印，不符合「避免重複及浪費」的注意事項。
~僅勾選數字最小的房號進行列印，其餘皆不勾選#流程規定是先全選後再「取消最大號」，而非僅勾選最小號。
~點選全選後，將數字最小的房號取消勾選後列印#操作指南明確指出應取消的是「數字最大」的房號。
~不需全選，手動逐一勾選需要的房號即可#系統操作建議先全選再過濾，以提高效率並減少遺漏。
}

::OP-FO-25-Q03::若訂單來源為 Agoda、Trip 或 Expedia 等串單至德安系統，在製作旅客住宿登記卡時，備註欄位應如何填寫？ {
=備註寫「已付」，且不可寫出金額#根據補充說明1，OTA串單需標註已付狀態，但為保護商業條件，嚴禁寫出實際金額。
~備註寫「已付」，並註明訂單上的總房價金額#補充說明特別強調「不可以寫出金額」，以免與客人實際支付給平台的金額產生認知落差。
~備註寫「未付」，待客人到店後確認收款#這些通路通常為線上預付，應寫「已付」而非「未付」。
~備註寫「已授權」，確保信用卡資訊有效#串單訂單已由平台收款，應標記「已付」，「已授權」通常用於前檯預先授權之訂單。
~備註需詳細寫出房價、稅金與平台名稱#流程規定只需寫「已付」且隱藏金額，過多細節不符合標準流程。
}

::OP-FO-25-Q04::在填寫「房況表」時，關於顏色標記與符號的使用，下列哪一項敘述符合標準流程？ {
=當日退房的房號畫綠色螢光筆，留房畫紅色螢光筆#這是房況表視覺化管理的核心規則，用以區分今日釋出房間與保留房間。
~當日退房的房號畫紅色螢光筆，留房畫綠色螢光筆#這與規定的顏色定義完全相反，會導致房務與櫃檯溝通錯誤。
~當日預進的房號應使用藍色原子筆圈起#規定要求當日預進應使用「鉛筆」圈起，以便後續異動或擦改。
~續住房間只需在房號旁寫上 I/H，不需寫日期#規定要求續住房需寫上「I/H + 退房日期」，以便掌握該房後續狀態。
~外國人使用的房間應畫紫色螢光筆註記#流程規定外國人房間應「註記星號」，而非使用特定顏色螢光筆。
}

::OP-FO-25-Q05::為了確保住客安全與居住品質，若房況許可，應盡量避免安排哪些類型的房間？ {
=避難逃生設備房間（如 202, 305）以及角間#作業指導書中明確列出 202, 305, 606, 807 及角間為盡量避免安排的房間。
~靠近電梯的房間以及低樓層房間#低樓層反而是多間房間團體住客的首選，電梯旁則未在禁止名單中。
~高樓層景觀房以及電梯正對面的房間#高樓層通常是特殊需求，而非需避免安排的對象。
~位於走廊盡頭的房間以及 SF 特殊房型#走廊盡頭雖常被稱為角間，但重點應在於是否包含避難設備或特定編號。
~剛維修完成的房間以及備品較多的房間#維修完畢的房間若經確認即可使用，並非流程中列出的避開對象。
}

::OP-FO-25-Q06::製作旅客住宿登記卡時，若訂單備註顯示已進行「預先授權」，應在 R 卡備註欄填寫什麼？ {
=已授權#流程明確規定依據訂單狀況判別，若有預先授權則需在 R 卡備註寫上「已授權」。
~已付#預先授權並不代表實際收款扣款完成，因此不能寫已付。
~未付#已有授權額度，與完全未提供支付保證的「未付」狀況不同。
~待核款#此非流程內標準化的備註用詞。
~信用卡結清#結清代表已完成交易，授權僅是凍結額度，意義不符。
}

::OP-FO-25-Q07::針對 602 房型，填寫房況表時需要特別注意什麼？ {
=需確認當日要做 SF 或 DD，並填寫於表內#指導書指出 602 為特殊房型，需確認當日房態設定（SF 或 DD）。
~一律安排給外國籍旅客使用並打星號#註記星號是針對所有外國人房間的規則，非僅限 602。
~必須手動更改為併床模式#併床模式通常是針對 CT（兩小床）房型的調整，非 602 的固定要求。
~禁止排入多筆同名訂單的住客#流程中並無此項限制。
~需在房號上畫紅色螢光筆標示為特殊房#紅色螢光筆代表「留房」，並非特殊房型的專屬標記。
}

::OP-FO-25-Q08::當客人預訂多間房間時，排房策略應如何調整以減少對其他住客的影響？ {
=盡量排在低樓層且同一樓層，方便上下樓並集中音量#文件建議多間房者排低層與同層，既方便住客移動，也能避免音量分散影響整棟旅客。
~將房間分散於各樓層，以稀釋該團體的音量#這會增加住客在走廊串門子的機率，反而可能干擾更多樓層的客人。
~全部安排在靠近逃生梯的房間#逃生設備房間（如 202, 305 等）應盡量避免安排住客。
~優先安排在高樓層以遠離大廳噪音#低樓層是針對多間房住客的建議策略，高樓層通常保留給有特殊需求者。
~依照訂單編號隨機分配房號即可#排房需考慮注意事項中的樓層分配原則，不可完全隨機。
}

::OP-FO-25-Q09::關於房況表上的備品註記，下列作業方式何者正確？ {
=在備註格中，寫上當日入住需提供的備品（如床圍）或退房需收回的備品#房況表是櫃檯與房務聯繫備品的重要工具，需清楚註記借出與回收項目。
~僅需寫下入住需增加的備品，退房不需記錄收回項目#流程規定退房也須收取備品（如澡盆、板凳）並做紀錄。
~若為留房，則不需寫上須收回的備品資訊#文件中特別註明「若有留房也須寫上須收備品」。
~備品數量不需寫出，只需寫出品名即可#流程範例中明確寫到「備品名及數量 (EX\:床圍*1)」。
~所有備品資訊皆應寫在 R 卡上方，而非房況表內#備品需求應寫在「房號的備註格中」，這指的是房況表的操作。
}

::OP-FO-25-Q10::若客人於官網訂購高鐵聯票（依據 2026/5/2 更新前的參考做法），R 卡備註金額應如何處理？ {
=備註金額只需寫出「總房價」，勿包含高鐵票金額#為了房費帳務清晰，備註金額僅標示房價部分，避免與交通費混淆。
~備註寫出「房價 + 高鐵」的總合金額#補充說明 2 特別強調「勿寫房價 + 高鐵的總金額」。
~備註寫上「高鐵聯票」，但不需寫任何金額#規定要求仍須寫出「總房價」的金額。
~需分別列出房價金額與高鐵票金額#規定僅要求寫出「總房價」，不需拆解列出高鐵金額。
~此類訂單一律標記為「已授權」，不寫金額#高鐵聯票通常包含款項收付，金額標註有其特定格式規定。
}
