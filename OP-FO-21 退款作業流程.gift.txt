$CATEGORY: OP-FO-21 退款作業流程

::OP-FO-21-Q01::當客人透過「電話訂房」並已完成匯款，後續因故需辦理退款時，櫃檯人員應請客人提供哪一項關鍵資訊以利作業？ {
=提供存摺封面照片#根據流程規範，電話訂房或ATM轉帳之退款，必須索取收款人存摺封面以確保帳號資訊正確。
~提供原匯款收據正本#雖然收據可用於對帳，但退款流程中明確要求的是存摺封面以便進行轉帳作業。
~提供身分證正反面影本#流程中並未要求提供個人身分證件，主要核對的是收款帳戶資訊。
~提供原訂房確認簡訊#訂房確認簡訊僅用於確認訂單，無法作為退款撥款的依據。
~提供信用卡正反面影本#電話訂房且已匯款的情況下應採銀行轉帳退款，而非信用卡刷退。
}

::OP-FO-21-Q02::針對退款作業的工作天數，櫃檯人員在與客人溝通時，應如何說明標準作業時間？ {
~告知需要 3 至 5 個工作天#此天數低於標準作業流程所訂定的時間範圍。
=告知需要 7 至 14 天（不含假日）#作業流程明確規定退款工作天為 7 至 14 天，且特別註明不含假日。
~告知需配合財務每月固定結算日撥款#雖然財務有其流程，但標準話術中應給予具體的天數範圍。
~告知次月底前會完成退款#流程中並非以「次月底」作為承諾，而是以天數計算。
~告知約需 21 個工作天#此天數過長，可能引起客人不滿，並非標準流程規範的時間。
}

::OP-FO-21-Q03::關於退款手續費的說明與收取標準，下列敘述何者符合現行規範？ {
~無論何種退款方式，皆一律免收工本費#並非所有方式都免收，僅限信用卡退款免收工本費。
~退款會收取 10 元的行政處理費#金額不符合規範中提到的 15 至 30 元範圍。
=銀行轉帳退款會收取 15 至 30 元的手續費#根據話術規範 1-2，應告知退款會產生 15 至 30 元的手續費。
~手續費一律由飯店全額吸收#除非特定情況，否則標準流程中會告知客人將扣除手續費。
~手續費固定為退款總金額的 $10\\%$#手續費為固定級距金額，並非按退款比例計算。
}

::OP-FO-21-Q04::若客人在預定住宿日期的「前 2 天」提出取消訂房申請，根據最新的取消規定，其訂金退還比例為何？ {
~退還 $100\\%$ 訂金#前 3 天取消才可退還 $100\\%$。
~退還 $80\\%$ 訂金#現行規範中並無 $80\\%$ 的退款級距。
=退還 $50\\%$ 訂金#最新取消規定指出，預定住宿日前 1-2 天取消者退還 $50\\%$。
~退還 $30\\%$ 訂金#現行規範中並無 $30\\%$ 的退款級距。
~不予退還任何金額#當日取消才會沒收全額訂金。
}

::OP-FO-21-Q05::若客人在「訂房網站（平台）」預訂且欲於「當天」取消，櫃檯人員應如何協助或告知相關權益？ {
~由櫃檯直接在後台退回現金給客人#平台訂單必須透過平台聯繫，飯店不能直接進行退款操作。
=告知會沒收全額，但可於三個月內使用#針對平台當日取消，規範註明會沒收全額，但提供三個月內可使用的彈性。
~告知平台訂單一旦取消即無法再使用#雖然沒收全額，但規範中有提到三個月內使用的備註。
~請客人聯繫平台，飯店可協助爭取全額退款#規範中並未承諾當天取消可全額退款。
~告知當天取消可退還 $50\\%$ 訂金#當天取消屬於沒收訂金的情況，不適用比例退還。
}

::OP-FO-21-Q06::在 UOF 系統填寫「B0.02 訂房退款申請單」時，若「退款類別」勾選為「信用卡退款」，則「是否酌收工本費」一欄應如何處理？ {
~依據退款金額大小決定是否收取#工本費收取與金額大小無關，與退款方式有關。
~一律勾選「是」#信用卡退款是唯一明確標註不收取工本費的項目。
=一律勾選「否」#申請單表單備註明確指出，如為信用卡退款，一律「不」收取工本費。
~視客人取消原因是否具備正當性而定#行政費用的收取是依據支付渠道的特性，而非取消原因。
~由財務部審核後再自行勾選#申請人應於填寫表單時即根據規則完成勾選。
}

::OP-FO-21-Q07::在「官網訂房已付款」的情況下，若客人表示無法在官網找到訂單進行取消，櫃檯人員應執行的正確步驟為何？ {
~直接在德安系統點選取消即可#官網訂單需先在 web 介面處理，以確保金流與資訊同步。
~請客人自行聯繫發卡銀行辦理爭議帳款#這不是標準的退款作業方式，且會造成金流混亂。
=至 web 手動取消，並在處理說明打上原因，確認德安是否下傳成功#這是官網訂單無法自行取消時的標準補救SOP。
~請客人等到入住日當天未到，系統會自動辦理退款#No show 通常會沒收全額，且不會自動觸發退款流程。
~告知官網訂房若未自行取消則無法受理退款#飯店應協助客人完成手動取消流程，而非拒絕受理。
}

::OP-FO-21-Q08::在 UOF 退款申請流程中，完成表單內容填寫後，必須新增的附件圖片「不包括」下列哪一項？ {
~德安訂金明細表#這是 UOF 流程第 7 點明確要求的必備附件。
~存摺封面（若有需要時）#涉及銀行轉帳退款時必須檢附存摺封面。
~信用卡刷卡簽單#表單下方標註了附件可包含信用卡刷卡簽單。
=客人入住時的登記表 (Registration Card)#退款作業主要核對訂金與帳戶，不需檢附入住登記表。
~靈知訂房作業&訂金收款截圖#表單下方明確標註可作為附件的選項之一。
}

::OP-FO-21-Q09::針對「電話訂房已匯款」的取消作業，在德安系統中的正確操作路徑為何？ {
=訂房管理 -> 輸入單號查詢 -> 開啟 Profile -> 點選取消單筆並存檔#此為德安系統取消電話訂房的標準作業四步驟。
~櫃檯接待 -> 房務管理 -> 選擇客房狀態 -> 執行訂單作廢#取消訂單應在「訂房管理」功能模組下操作。
~出納管理 -> 訂金查詢 -> 選取訂單 -> 執行退款存檔#這是退款帳務操作，而非取消訂單狀態的操作。
~訂房管理 -> 輸入姓名查詢 -> 開啟交易明細 -> 點選退款並存檔#取消訂單應點選「取消單筆」，而非直接在交易明細處理。
~系統設定 -> 資料維護 -> 批次取消 -> 存檔離開#此路徑為系統管理使用，非一般櫃檯取消單筆訂單的流程。
}

::OP-FO-21-Q10::完成 UOF 退款申請單的填寫與圖片新增後，最後一個需要在德安系統執行的動作是什麼？ {
~直接將訂單從德安系統中刪除#訂單不可隨意刪除，應保持取消狀態以供稽核。
=在德安備註欄位打上「填退日期」#退款申請流程第 9 點明確要求須於德安備註打上填單日期。
~列印退款單並由主管簽名#UOF 為電子簽核流程，不需額外紙本列印簽名。
~立即執行結帳作業將餘額歸零#退款需經財務審核，不應由櫃檯人員先行在系統結帳歸零。
~將訂單狀態更改為「已退款」#德安系統通常只有取消狀態，退款進度由 UOF 與財務系統控管。
}
