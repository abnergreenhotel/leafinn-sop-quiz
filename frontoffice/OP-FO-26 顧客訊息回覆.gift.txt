$CATEGORY: OP-FO-26 顧客訊息回覆

::OP-FO-26-Q01::當你在上午 10\:00 收到一位客人在官方 LINE 詢問訂房房價時，根據服務標準，最晚應在何時完成第一次訊息回覆？ {
~10\:15 前#雖然更快速的回覆能提升滿意度，但標準規範的時限並非 15 分鐘。
=10\:30 前#為了確保顧客感受良好的服務時效，規範要求所有管道的訊息均需在 30 分鐘內回覆。
~11\:00 前#一小時的回覆時間已超過標準要求的時限，可能導致顧客流失。
~中午 12\:00 前#此時限過長，不符合飯店對於即時訊息溝通的標準作業規範。
}

::OP-FO-26-Q02::在處理 OTA（如 Booking.com）的訂單時，發現系統沒有自動串單，且該訂單在 WEB 後台顯示為「可下傳」狀態，此時應如何處理？ {
~直接在德安系統手動建立一筆新訂單#若訂單狀態仍可執行系統下傳，應優先使用系統功能以確保資料正確性。
~在 WEB 後台點擊「下傳」後，立即確認德安系統#系統處理資料需要時間，點擊後應等待五分鐘再行確認才較為準確。
=在 WEB 後台按「下傳」並等待五分鐘，再確認資料是否正確#當系統未自動串單但狀態允許時，手動觸發下傳並預留系統作業時間是正確的標準程序。
~立即回報主管處理，不要操作後台#此狀況屬於可排除的作業異常，應先嘗試依流程操作下傳，若仍失敗才需回報。
}

::OP-FO-26-Q03::客人在 FB 私訊反映昨晚入住時房間清潔不理想，並附上照片表達不滿，此時第一線夥伴的首要動作為何？ {
=先向客人道歉並將此客訴轉交給主管#客訴案件具高度敏感性，應依照職權分工轉由主管審閱或處理，以確保回應妥當。
~自行回覆願意補償客人下次入住優惠#補償方案通常涉及職權歸屬，不應在未回報主管的情況下私自承諾。
~請客人直接撥打飯店電話聯繫現場櫃檯#這會增加客人的負負擔，應由內部轉派相關負責人員主動處理此訊息。
~將訊息轉傳給行銷部處理#行銷部主要處理抽獎與合作，服務品質相關的客訴應優先由主管掌握。
}

::OP-FO-26-Q04::在檢查 OTA 訂單需求時，發現客人在備註中要求一組「澡盆」和「床圍」，但目前館內備品庫存已全數借出，應如何操作？ {
~先在德安系統註記，等客人入住時再告知已借完#現場才告知會造成客人期望落差，應儘早進行雙向溝通。
=直接在該 OTA 平台後台發送訊息告知客人備品已無庫存#針對特殊需求無法滿足的情況，必須在第一時間透過平台聯繫客人，提供良好的預期管理。
~確認訂單金額無誤後，不需回覆備品問題#忽略客人的特殊需求備註會導致服務不周，屬於嚴重的作業疏失。
~詢問主管是否可以去其他館別借調後再回覆#SOP 規定若當下已無備品，應「第一時間」先在後台告知顧客現狀。
}

::OP-FO-26-Q05::夥伴正在處理電子郵件，發現有一封信是政府機關寄來的正式公文，這類信件的處理程序是什麼？ {
~回信請對方寄送紙本公文至飯店#這會延誤公務處理時間，且電子公文已有正式效力。
~自行影印存檔後，放置於公文夾內#公文涉及行政決策，不可由第一線人員自行存檔結案。
~將信件內容轉寄給行政窗口即可#行政窗口未必具備決策權，SOP 明確規定公文必須上呈。
=立即將該信件轉給主管審閱#不論是總部信件還是政府公文，涉及職權與正式規範者皆需由主管審核。
}

::OP-FO-26-Q06::在處理 Google 或 Agoda 等平台的顧客評論時，下列哪一項是規範要求的必要動作？ {
~只要回覆內容禮貌，不需要另外紀錄#僅回覆而不紀錄會導致公司無法統計並追蹤顧客意見趨勢。
=需將評論內容登記在「顧客意見表」中#登記意見表是為了進行後續的品質管理與數據統計，是回覆評論的重要步驟。
~一週檢查一次各平台是否有新評論即可#評論檢查需具備時效性，規範要求應「每日」進行確認。
~負面評論才需要紀錄，正面評論直接回覆即可#不論正面或負面意見，皆應統一登記以利全面性的服務優化參考。
}

::OP-FO-26-Q07::夥伴正在處理 OTA 訂單下傳，突然發現發生了「爆房」（超賣）的情況，正確的應對邏輯為何？ {
~先嘗試聯繫客人取消訂單，減少損失#擅自要求客人取消會引發嚴重爭議，必須先確認內部應對策略。
~等到交接班時，再請下一班夥伴處理#爆房屬於緊急突發狀況，延後處理會導致問題擴大並增加排解難度。
=立即通知主管，確認後續處理流程#爆房涉及訂單調度與潛在的轉單成本，必須第一時間向上呈報以利緊急決策。
~手動建立訂單並備註「爆房」，等主管上班再說#僅作備註而未立即通知，無法及時阻斷持續超賣的可能性。
}

::OP-FO-26-Q08::某公司負責人透過 FB 私訊詢問長期合約優惠與專案合作，這類訊息應轉給哪個部門？ {
=行銷部#涉及企業合作、行銷抽獎等對外業務擴展事宜，標準流程係轉交行銷部統籌。
~財務部#財務部處理核銷與金流，並非初期業務接洽與方案設計的窗口。
~房務部#房務部負責清潔與客房維護，不涉及合約談判或企業合作業務。
~一般訂房組#訂房組處理常規訂單，企業專案合作需由專業行銷或業務單位先行規劃。
}

::OP-FO-26-Q09::顧客在深夜透過 Email 諮詢非常複雜的團體訂房需求，需要由專業的訂房主管核算報價，但主管已下班，此時應如何處理？ {
~等隔天主管上班核算出價格後再回信#這會違反 30 分鐘內回覆一次訊息的標準時效規定。
=先回信告知顧客已收到訊息，並說明將由專人後續回覆#若需晚點回覆或由專業人員處理時，必須先行告知顧客，以符合回覆禮儀並展現負責態度。
~先隨便預估一個價格回覆，等明天再修正#錯誤的價格資訊會引發嚴重的後續合約糾紛，應避免提供不確定的資訊。
~不回覆 Email，直接撥打電話給客人說明#顧客選擇使用 Email 溝通通常有其偏好，應先在原管道回覆以確保紀錄完整。
}

::OP-FO-26-Q10::在操作德安系統處理 OTA 訂單時，若訂單有特殊需求（如：需要澡盆），除了加上備註外，還有哪項重要的操作？ {
=需至德安前台扣除該備品數量#為了確保存貨管控精確，於訂單標註需求的同時，應在系統內同步扣除相對應的庫存數量。
~將需求寫在白板上提醒下一班#白板紀錄僅為輔助，系統化的扣帳與備註才是標準化的正確流程。
~僅需在德安系統加上文字備註即可#只加備註而未同步扣除庫存，會導致備品管控失準，產生超借風險。
~拍下訂單備註傳送到主管 LINE 群組#一般備品需求應落實於系統作業流程中，不應以此行政細節頻繁干擾群組訊息。
}
