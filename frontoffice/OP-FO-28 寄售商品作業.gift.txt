$CATEGORY: OP-FO-28 寄售商品作業

::OP-FO-28-Q01::當客人選購完寄售商品並來到櫃檯結帳時，夥伴應優先執行哪一項口語確認動作？ {
=確認商品的名稱、數量以及最終結帳價格#透過口頭向客人核對購買內容與金額，能有效減少後續因資訊不對稱產生的糾紛。
~詢問客人是否需要包裝或紙袋#雖然是服務細節，但在作業流程中，核對商品與金額是確保交易準確性的首要步驟。
~直接請客人出示房卡以確認身份#部分合作商品可能僅收現金或需即時入帳，應先確認商品資訊而非優先要求房卡。
~告知客人寄售商品一旦售出概不退換#這並非標準作業流程中的首要確認話術，且容易給客人負面的服務感受。
}

::OP-FO-28-Q02::在處理商品銷售時，除了在德安系統入帳外，關於「扣除庫存」的正確操作流程為何？ {
~在紙本登記簿上記錄銷售品項與日期#目前的作業方式已數位化，使用紙本紀錄不符合現行的雲端管理規範。
=開啟資料夾，搜尋品項並輸入購買數量、金額與日期#透過雲端資料夾即時更新，能讓管理人員隨時掌握商品目前的庫存水位。
~等到班末交接時再統一更新庫存報表#延遲更新庫存可能導致帳務與實物不符，應於交易發生時即時處理。
~僅需在德安系統點選「扣庫存」按鈕即可#德安系統主要處理帳務，庫存明細需另外在指定的雲端空間進行更新。
}

::OP-FO-28-Q03::若有客人欲購買「無拘無塑」展覽合作商品，櫃檯夥伴在處理收付款時應注意什麼？
目前配合的寄售商品都可以用任何方式付款 {
=目前配合的寄售商品都可以用任何方式付款，若之後配合展覽商品有限定現金再依當時規定為主#這類特定合作商品有支付限制，統一收取現金能簡化後續與廠商的對帳作業。
~鼓勵客人使用信用卡，以利系統自動對帳#這與該類商品的特殊支付規定相抵觸，會造成財務結報上的困擾。
~請客人直接將款項交給展覽負責人#櫃檯應作為統一收支窗口，隨後再錄入系統並備註賣出名稱以利核對。
~若客人沒有現金，可先將費用掛在房帳中待退房支付#合作商品需即時結帳且僅收現金，不應採用房帳抵付方式處理。
}

::OP-FO-28-Q04::在德安系統中執行入帳操作時，找到該名旅客後，正確的操作路徑為何？ {
=按右鍵點選「旅客帳處理」→「入帳」→「消費項」#這是系統處理個人雜項消費的標準路徑，確保款項正確歸屬到該客帳。
~直接點選「快速結帳」→「商品銷售」#德安前台系統的標準入帳流程是透過旅客帳處理，而非快速結帳按鈕。
~點選「訂房管理」→「變更房價」#商品銷售屬於額外消費，與房價變更無關，不應修改房價欄位。
~點選「財務管理」→「雜項收入」#在接待管理模組中直接處理客帳是較為即時且正確的操作方式。
}

::OP-FO-28-Q05::若在盤點或銷售過程中，發現寄售商品出現嚴重損壞導致無法販售，夥伴應如何處理？ {
=告知客人商品損毀，請其直接照價賠償，並視情況請主管處理#針對人為導致無法販售的損失，應維持原價賠償原則以維護合作廠商權益。
~將損壞商品報廢，並在雲端資料夾註記「耗損」#寄售商品的所有權屬於廠商，若無授權不得私自報廢，且需追蹤損失責任。
~給予客人折扣，半價將損壞品賣出#商品已損壞且無法正常販售時，不應隨意降價求售，應按損壞處理程序辦理。
~自行修復商品後，放回架上繼續販售#修復後的商品可能存在瑕疵，不符合販售標準，且可能引發後續客訴。
}

::OP-FO-28-Q06::關於銷售寄售商品後的發票開立，下列哪一項敘述符合目前的作業規範？ {
~所有寄售商品都不能使用載具#作業指導書中明確指出，電子發票列印是特定分館才需要的作業。
=結帳後需點選確定以列印電子發票#各館別的稅務處理設備與流程不同，目前僅規範二館需執行此步驟。
~發票開立需在每月底統一由會計開立並寄送給客人#商品銷售通常為現場即時交易，應於結帳時確認發票處理流程而非事後寄送。
~寄售商品屬於代售性質，館方一律不開立發票#即使是寄售商品，在飯店系統入帳後仍有對應的財務流程需遵循，包括特定館別的發票開立。
}

::OP-FO-28-Q07::在「月結」作業中，主管的主要工作內容是什麼？ {
~親自向購買過商品的客人進行售後滿意度電訪#月結工作的重點在於帳務與庫存的結轉，售後服務並非月結的核心流程。
~自行決定下個月要引進的新商品種類#月結是指對既有銷售紀錄的彙整，與新商品的開發採購分屬不同階段。
=與廠商進行盤點核對，結算當月銷售金額並提供會計計算#透過三方（廠商、現場數據、會計）的核對，確保寄售帳務的準確性。
~直接將當月收到的現金寄回給合作廠商#款項需經過會計審核與正式的財務結報程序，不可由現場主管私自寄送現金。
}
