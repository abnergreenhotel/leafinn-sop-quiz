$CATEGORY: OP-FO-29 夜間稽核SOP

::OP-FO-29-Q01::在進行當日帳務總檢時，若發現有一筆信用卡授權單據遺失，且系統顯示該筆帳款未結，最正確的處理方式為何？ {
~為了不延誤過夜時間，先將該帳務直接結轉，等隔日早班再補單。#異常帳款若未經釐清直接帶入過夜，會導致後續財務查帳困難且帳務不連續。
=應先釐清該筆帳款的來源並確認單據去向，於過夜前完成核對。#確保所有單據齊全並釐清異常案件，是維護當日帳務正確性並順利進行稽核的關鍵步驟。
~直接手動沖銷該筆帳款，將差額記錄在當日損益中。#手動沖銷而未經核實會造成實際營收與系統數據不符，違反會計稽核原則。
~僅需在交班表註記「單據遺失」，不需尋找單據即可進行下一步。#註記雖重要，但 SOP 要求必須先確認單據齊全並釐清異常，而非僅是口頭或書面交代。
}

::OP-FO-29-Q02::夜間稽核人員核對卡機總額與系統報表時，發現實收金額與報表數字不符，此時應優先執行的動作是什麼？ {
=先找出導致金額不一致的原因，確認無誤後再進行結轉。#確認每日收入資料的一致性是結轉的前提，避免將錯誤數據滾動至隔日帳務。
~以卡機金額為準，直接修改系統報表的數字使其符合。#未經查證就強行平衡帳務，會掩蓋實際發生的操作錯誤或金流漏洞。
~先行執行系統結轉，待財務部後續通知後再補正。#系統結轉後數據會存檔，若帶著錯誤結轉會大幅增加財務同仁後續核對的行政成本。
~將該筆差額掛在「待處理帳項」，留給早班同事核對現金。#夜間稽核的職責包含確認當日收入一致，不應在未嘗試尋找原因的情況下推給下一班。
}

::OP-FO-29-Q03::在檢查房態時，發現有一間房在系統中顯示「待入住」，但目前已是凌晨時分且查無入住紀錄，此動作對隔日作業有何直接影響？ {
~沒有影響，只需等待早班人員聯絡客人即可。#錯誤的房態會產生虛假的可用房資訊，直接干擾隔日的排房規劃。
=會直接影響隔日的排房作業與接待流程。#正確的過夜房態（在住房、未到店、空房）是確保隔日櫃檯接待與排房邏輯正確的基礎。
~僅會影響當日的早餐人數清點，不影響房況。#房態異常的影響層面不僅限於早餐，更核心的是影響房間的調度與銷售狀態。
~只需確認該房已清潔，系統狀態不正確並不妨礙排房。#系統房態若與實際不符，會導致排房人員誤判房間可用性，造成接待衝突。
}

::OP-FO-29-Q04::關於「德安系統」夜間稽核的啟動時機，下列哪項敘述最符合 SOP 要求？ {
~只要到了凌晨 $1\:00$ 即可直接啟動，不必理會其他事務。#系統稽核是最後一步，必須在確認常態事務（如早餐人數、沖銷帳）完成後才可啟動。
~只要確認信用卡與現金金額正確，即可先行啟動系統稽核。#除了金流，房態確認與早餐人數清點等常態性事務也是啟動稽核的必要條件。
=確認早餐人數清點、沖銷帳與排房等常態事務均已完成且無異常後再執行。#確認所有前置條件完整後再啟動系統稽核，可避免系統作業中途出錯且難以修正。
~系統稽核完成後，再開始處理當日的早餐人數清點與排房作業。#順序錯誤，常態事務必須在系統執行稽核前確認完成，以確保數據一併結轉。
}

::OP-FO-29-Q05::在填寫「每日交班稽核表」時，有關報表留存與整理的原則，何者是正確的？ {
=報表留存順序需固定，以利財務同仁後續查核。#固定的報表留存與整理邏輯能提升後續財務核對的效率，減少溝通成本。
~隨意擺放即可，只要有將所有單據放入資料夾中即可。#缺乏組織的報表存放會增加後續查帳的困難，不符合財務規範。
~只需填寫總金額，細節的行動支付或信用卡明細不需要分類。#交班稽核表需涵蓋各類支付（信用卡、現金、行動支付）營收，分類不清會導致對帳異常。
~報表正本應交由夜班自行保管，不需交接給財務。#報表與稽核表的核心功能就是提供財務核對，必須按流程留存與轉交。
}

::OP-FO-29-Q06::若夜間稽核作業中遇到特殊無法處理的案件（如：金額異常無法即時查出原因），應如何進行交接？ {
~若早班主管沒問，就不必特別說明，以免被究責。#隱瞞異常會導致問題擴大，違反明確交接的原則。
~口頭告訴下一班同仁即可，不需浪費時間書寫記錄。#口頭交接極易遺忘或產生誤解，SOP 明確規定不可只口頭帶過。
=將事件記錄下來，並明確交接給隔天早班主管處理。#特殊情況必須透過書面記錄明確轉達給主管處理，確保責任歸屬與問題解決。
~直接將該筆異常帳務刪除，使當日報表看起來平衡。#刪除異常數據是嚴重的違規行為，會造成帳務缺漏與法律風險。
}
