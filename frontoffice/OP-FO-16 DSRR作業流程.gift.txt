$CATEGORY: OP-FO-16 DSRR作業流程

::OP-FO-16-Q01::當一位客人的房間是由其友人代訂時，若客人在辦理入住時好奇詢問房間費用，根據作業規範，櫃檯人員最專業的處理方式為何？ {
~直接告知當初訂房人支付的正確金額#隨意透露他人代訂的價格可能觸犯隱私，且若代訂者有特殊折扣，直接報價會造成雙方困擾。
=婉轉請客人直接詢問原代訂人#這能確保房價資訊由訂房人親自與住客溝通，避免櫃檯人員介入代訂者與住客間可能的金錢協議。
~主動告知飯店當日的現場掛牌房價#告知現場價可能導致客人與代訂價格做比較，產生不必要的誤會或客訴。
~請客人自行上網搜尋該房型的定價#這種做法會顯得服務態度消極，且網路價格變動大，無法準確回應客人的疑問。
}

::OP-FO-16-Q02::對於「旅行社」代訂的訂單，當住客試圖詢問房價時，櫃檯人員在執行 DSRR 流程時應遵守哪項關鍵原則？ {
~提供合約價並提醒客人不可外傳#即使提醒不可外傳，透露合約價仍違反了 DSRR (不顯示房價) 的核心操作準則。
~建議客人改用現場價格重新訂房#這不僅破壞了與旅行社的合作關係，也可能讓客人感到被刁難。
=婉告無法提供房價，且儘量不告知現場房價#避免客人將旅行社代訂價格與飯店現場報價進行比較，是保護合作渠道價金隱私的重要手段。
~直接列印系統中的訂單明細交給客人#系統明細通常含有敏感的報價資訊，直接交付會完全違背 DSRR 的規範。
}

::OP-FO-16-Q03::當客人透過 OTA (如 Agoda, Booking.com 等) 預訂 DSRR 房型，並在櫃檯要求確認付款金額時，同仁應如何運用標準話術應對？ {
=告知系統內「看不到」訂房金額#這是一種專業的推託術語，能讓客人在不感到被拒絕的情況下，理解飯店無法提供該平台的成交資訊。
~要求客人提供信用卡刷卡簡訊以利核對#這會增加客人的操作負擔，且與 DSRR 要求飯店保持價格保密的立場無關。
~解釋因為平台會抽成，所以金額不固定#過度解釋後台的商業邏輯是不必要的，且可能引發客人對平台佣金的負面聯想。
~告知客人系統故障，目前無法查詢#謊稱系統故障會損害飯店的專業形象，且不是長久的解決方法。
}

::OP-FO-16-Q04::一位入住 DSRR 房型的商務客要求櫃檯開立「住宿證明」以利回公司報帳。在開立此證明時，下列哪項做法符合最新修訂的注意事項？ {
~在證明上詳細列出房費與稅金明細#DSRR 訂單的初衷即為隱藏房價，在證明中列出明細會使房價保密功能破功。
~僅於備註欄位寫下客人支付給平台的總額#即便在備註欄撰寫，依然揭露了具體金額，不符合保密規範。
=明確告知住宿證明中「不可開立房費」#為了符合 DSRR 的原則，證明文件僅能確認入住事實，不能涉及具體的房費金額。
~請客人在空白處自行填寫金額後加蓋店章#這屬於嚴重的行政疏失與偽造文書風險，絕對禁止發生。
}
