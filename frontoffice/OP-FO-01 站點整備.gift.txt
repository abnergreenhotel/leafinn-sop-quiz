$CATEGORY: OP-FO-01 站點整備

::OP-FO-01-Q01::在開始接手櫃台任務前，你發現櫃台桌面有一份上一班留下的「今日客訴案件摘要」。根據作業標準，你應該如何處理這份文件？ {
~將文件夾在交班紀錄簿中，方便隨時翻閱跟進。#雖然夾在紀錄簿中方便，但仍有讓顧客在辦理手續時不慎窺見敏感資訊的風險。
=將其收納至抽屜或指定隱蔽處，確保櫃台表面無敏感文件外露。#維持櫃台整潔並避免敏感文件外露，是確保顧客隱私與專業形象的關鍵步驟。
~留在桌面上明顯處，提醒自己待會一定要優先處理。#這違反了櫃台環境檢查中「無敏感文件外露」的標準，可能造成資訊外洩。
~立即將文件內容拍照記錄後丟棄，以保持桌面整齊。#隨意處置文件可能導致後續追蹤困難，且拍照並非標準的環境檢查流程。
}

::OP-FO-01-Q02::清點備品時，你發現房卡剩下的數量剛好等於今日預計入住的房間數。此時最理想的做法是什麼？ {
~因為數量剛好夠用，等今日住房高峰過後再補即可。#這會忽略了臨時損壞或增加需求的風險，不符合「當班即時補充」的原則。
~先觀察半小時，若入住進度超前再聯絡後勤補貨。#延遲補貨會增加高峰時段的操作壓力，違反了不可等到顧客使用時才處理的準則。
~將現有房卡全部發放出去，不足時再向其他樓層商借。#這屬於臨時應變而非標準整備，會導致服務流程不連貫並增加溝通成本。
=立即前往庫存區領取，將房卡補齊至規定的安全庫存量。#備品若低於安全庫存應立即補充，以確保在尖峰時段能順暢應對所有顧客需求。
}

::OP-FO-01-Q03::在執行設備功能確認時，你發現其中一台發票機卡紙且無法印出字跡。此時你應該怎麼做？ {
~將該發票機關機並貼上故障標籤，等有空再嘗試修復。#單純貼標籤無法解決營運需求，且未依照流程進行基本排除與回報。
=先進行基本排除測試，若無法修復則立即回報主管。#作業規定在發現設備異常時應先嘗試基本排除，並在尖峰時段前完成回報，以免影響營運。
~直接聯絡發票機維護廠商，請其派人到場維修。#在未回報主管且未進行基本測試前就叫修，可能造成不必要的成本或流程越權。
~先不管發票機，因為還有另外一台可以使用，等壞了再一併處理。#這違反了「發現異常需先回報，不可等到尖峰時段再處理」的注意事項。
}

::OP-FO-01-Q04::上一班的同事正準備交班，並表示「今天一切都很正常，沒什麼特別的事」。身為接班者，你的標準回應應為何？ {
~詢問同事今天是否有特別想分享的趣事，以舒緩交班氣氛。#這與業務交接核心無關，無法確保工作品質與資訊完整性。
~相信同事的判斷，直接簽署紀錄簿並讓對方下班。#未讀取交班紀錄不得接手櫃台，不可僅憑口頭表述而忽略書面詳閱。
=詢問「請問今天有什麼重要事項需要交接的嗎」，並詳讀紀錄簿。#這是標準話術要求，且規定必須詳讀未完成案件、特殊房況等資訊後方可接手。
~既然正常，就請同事先下班，自己等有客人來之前再看紀錄。#交班必須在接手前完成，事後才閱讀可能導致處理前班未竟事宜時出現疏失。
}

::OP-FO-01-Q05::在閱讀當日作業日誌時，你看到今日有維修工程進行，且前一班在交接簿提到一位 VIP 提早抵達。你該如何整合這些資訊？ {
~優先處理 VIP 的需求，至於維修工程等主管來了再問。#這忽略了不同來源資訊的關聯性，可能導致 VIP 房間受到工程噪音影響。
~將兩份紀錄分開判讀，確保每份紀錄都個別簽核完成。#SOP 明確指出不可只看單一紀錄，必須整合判讀才能掌握完整情境。
=整合判讀當日重點與交接事項，確認維修是否影響該 VIP 房況。#透過將日誌重點與交班事項整合，才能提前安排顧客需求並預防潛在衝突。
~因為日誌與交接簿資訊量太大，僅記錄在個人工作清單即可。#這會導致決策過程支離破碎，未能發揮整備階段資訊掌握的真正價值。
}

::OP-FO-01-Q06::你已經完成所有設備、備品與環境檢查並向上級回報。但在開站後的十分鐘，你發現大廳入口出現了落葉與灰塵，你的下一步是？ {
=立即補強清理，因為開站完成後仍需持續維持環境狀態。#SOP 指出開站完成不代表後續可忽略巡檢，值勤中若發現環境異常應持續維護。
~等到下一個整點巡場時間再去處理，現在應專注於櫃台接待。#環境異常應即時補強，而非死守固定的巡檢時段而忽略當下的凌亂。
~聯絡專責房務或清潔人員處理，因為自己已經完成「整備回報」。#雖然可尋求協助，但整備責任包含持續維持，不能以「已回報完成」為由推諉。
~不予理會，因為灰塵是在正式開站「之後」才出現的，不屬於整備範疇。#這是對 SOP 的誤解，整備工作的目的在於維持持續的優質服務環境。
}
