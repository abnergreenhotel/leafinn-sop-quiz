$CATEGORY: OP-OB-04 隱私資安

::OP-OB-04-Q01::一位自稱是房客熟識的朋友來到櫃檯，表示想要給房客一個驚喜，並請求您提供該房客的房號與住宿天數，您該如何處理？ {
~為了維持良好的服務品質並展現親切，核對該名朋友的身分證件後即可告知。#即使確認了詢問者的身分，在未經房客本人授權的情況下洩漏資訊仍會構成資安漏洞。
=婉拒提供資訊，並說明基於隱私保護原則，不可透露住客的房號與住宿細節。#嚴守隱私保護原則能確保住客的安全與信任，避免因第三方詢問而造成個資外洩。
~若該名朋友能提供房客的完整姓名與電話，即可視為已獲得房客授權，可以告知房號。#持有基本個資不代表擁有查詢隱私資訊的權限，這種判斷方式容易讓有心人士獲取敏感資料。
~私下查閱系統確認房客是否在房內，若房客在房內則請該位朋友直接上樓敲門。#引導訪客直接前往客房區不僅違反隱私規範，也可能造成住客驚嚇或產生安全風險。
}

::OP-OB-04-Q03::在設定與管理系統密碼時，下列哪一種行為符合公司規定的安全規範？ {
~將密碼寫在標籤紙上，黏貼在螢幕邊框或鍵盤下方，方便交接時查看。#將密碼放置於公開可見處會極大增加帳號被盜用的風險，違反基本的資安防護意識。
~將帳號與密碼透過通訊軟體的公開群組傳送，以便主管隨時查核。#在群組傳送機敏資訊容易導致外洩，且不符合交接應由主管或資訊窗口處理的流程。
=密碼應避免使用生日或電話等易猜測組合，且設定後不應記錄在他人看得到的地方。#使用複雜性密碼並妥善隱藏，能有效提高非法侵入的難度，保障系統與個資安全。
~為了方便記憶，多個不同的系統（如信箱與 PMS）皆使用同一組簡易連號密碼。#使用連號或跨系統共用密碼會產生連鎖風險，一旦一組密碼被破解，所有帳號都會暴露在危險中。
}

::OP-OB-04-Q04::櫃檯夥伴在處理退房流程中途，因住客緊急需要協助搬運行李而必須離開崗位，此時電腦畫面仍顯示住客名單與房卡資料，該夥伴應如何處理？ {
~直接前往行李房，因為只是離開幾分鐘，回來後再繼續處理即可。#暫時離開也可能導致路過的第三人窺視機敏資料，應隨時防範個資外流的可能。
~將電腦螢幕關閉即可，不需要執行登出或鎖定，以免回來時重新登入太耗時。#單純關閉螢幕並未結束系統作業，他人只需開啟螢幕即可接續操作或查閱資料，無實質保護作用。
~請隔壁正在忙碌的同事幫忙「盯著」螢幕，待回來後再繼續作業。#同事在工作時無法分心確保您的螢幕安全，且個資防護應由該崗位執行人員親自落實。
=必須先將系統登出或鎖定畫面，並確認紙本報表或名單沒有隨意放置於檯面上。#執行鎖定動作可確保在人員離開期間，無人能隨意接觸系統資訊，落實裝置管理規範。
}

::OP-OB-04-Q05::夥伴在館內看到某位知名 KOL 辦理入住，覺得很興奮，想要在個人的社群媒體分享工作生活，下列哪一項做法是正確的？ {
~可以拍攝系統上的房客名字與房號，但發布前先用貼圖把房號遮起來。#系統畫面本身屬於內部機敏資訊，無論是否遮蓋，均不得拍攝或發布至個人社交平台。
=嚴禁在個人社群媒體發布住客資料、內部系統畫面或館內未公開的營運數據。#明確區分個人生活與公務資訊界線，能防止公司敏感資訊在非授權管道流傳。
~若是有媒體或網友私訊詢問該名 KOL 的入住細節，應立即依據現有資訊詳細回覆。#未經授權的發言可能引發公關危機，面對外部詢問應統一轉由指定窗口或主管處理。
~只要不拍到 KOL 的長相，僅拍攝其行李吊牌或住宿登記表是沒有問題的。#行李吊牌與登記表同樣含有個資，拍攝並發布此類資訊已嚴重違反隱私資安規範。
}

::OP-OB-04-Q06::若在值班時發現系統帳號出現不明登入通知，或是懷疑公司配發的隨身碟內資料異常遺失，最正確的處理程序是什麼？ {
~為了避免擴散，立即刪除所有疑似異常的紀錄，並將設備格式化。#自行刪除紀錄會破壞事故現場，使後續資訊人員無法追蹤異常來源或評估損害程度。
=先保留現況，並立即通知主管與資訊窗口處理，不擅自進行後續操作。#保留現場並優先通報能讓專業人員及時介入，依循標準指令處理可降低事件影響範圍。
~因為情況不確定，應先觀察幾天，若持續發生異常再向上呈報。#資安事件具有時效性，延遲回報可能導致資料被進一步竊取或系統遭到更嚴重的破壞。
~私下詢問其他同事是否也有相同狀況，若大家都有，則判定為系統正常能量釋放。#資安判定應基於專業通報而非集體猜測，任何可疑跡象都應正式記錄並上報。
}
