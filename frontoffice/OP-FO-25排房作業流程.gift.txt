$CATEGORY: OP-FO-25排房作業流程

::OP-FO-25-Q01::當你在德安系統中發現同一位旅客「王大同」預訂了三間房間時，根據排房作業流程，你應該優先如何處理？ {
=將三間房間盡量安排在同一樓層，並優先進行排房。#SOP 指出同名多筆訂單應先行排房，且為了方便旅客與管理，應盡量排在同一樓層。
~為了分散客流量，應將這三間房間隨機分配到不同樓層。#將同名訂單分散樓層會增加旅客同行的不便，也不符合 SOP 優先排在同一樓層的原則。
~等當天所有單間預訂排完後，再處理這類多筆訂單的排房。#SOP 規定同名多筆訂單應「先行排房」，而非最後處理。
~直接交由系統自動分配，不需人工介入干預樓層。#SOP 要求人工確認特殊需求（如高樓層）並執行拖曳排房動作，以確保準確性。
}

::OP-FO-25-Q02::在房況許可的情況下，以下哪一個房號應該盡量避開，不優先安排給一般旅客入住？ {
=305 號房#SOP 明確列出 202、305、606、807 為避難逃生設備房間，除非房況吃緊，否則應避免安排。
~501 號房#SOP 僅提到應避免安排避難逃生設備房與角落房，501 並非文件中提到的限制房號。
~602 號房#602 是特殊房型，需確認 SF 或 DD 狀況，但並非 SOP 中列出應主動避開的避難設備房。
~708 號房#文件中未提到 708 號房有任何排房限制，可根據一般流程安排。
}

::OP-FO-25-Q03::執行「列印 R 卡」作業時，若遇到同批顧客有多個房號，為了避免浪費與重複，應該如何操作勾選清單？ {
=將同批顧客中數字最大的房號取消勾選後再列印。#SOP 規定在列印旅客登記卡時，應全選後取消勾選同批中數字最大的房號，以維持資料正確並減少浪費。
~直接點選「全選」並列印所有產出的 R 卡。#直接全選列印會導致多筆同名訂單重複列印，造成紙張浪費且不符合 SOP 規範。
~只勾選最小房號的訂單，其餘全部手動取消勾選。#SOP 的具體做法是取消「數字最大」者，而非僅選最小者，操作邏輯不符。
~先列印第一張，確認無誤後再手動一張張補印。#此做法效率過低，且 SOP 已有明確的大量處理勾選邏輯。
}

::OP-FO-25-Q04::若旅客是透過 Agoda 訂房且已在線上完成付款，在製作「旅客住宿登記卡（R卡）」的備註時，下列哪種標註方式正確？ {
=備註寫「已付」，但不寫出具體金額。#根據補充說明 1，若由 Agoda 等平台串單，備註需寫已付，且嚴禁寫出具體金額。
~備註寫「已付」，並註明訂單上的總金額以便確認。#SOP 特別強調「不可以寫出金額」，以免因平台價差或商業契約產生爭議。
~備註寫「未付」，等旅客現場出示憑證後再修改。#既然系統顯示已串單且為已付狀態，應正確標註為「已付」，否則會導致櫃檯重複收款。
~備註寫「已授權」，代表款項由平台擔保。#對於 Agoda 等已結算之串單，SOP 要求標註「已付」而非「已授權」。
}

::OP-FO-25-Q05::在填寫「房況表」時，關於房號標記的顏色規範，下列哪一項操作是正確的？ {
=在當日退房的房號上畫綠色螢光筆。#SOP 規範當日退房使用綠色螢光筆標記，以便快速識別空房進度。
~在當日預進的房號上畫紅色螢光筆。#SOP 規定當日預進房號應用「鉛筆圈起」，而非螢光筆。
~在續住的房號上畫紅色螢光筆，不需寫字。#紅色螢光筆適用於「留房」，且續住房需另外加註 I/H 與退房日期。
~在所有預計入住的房號上畫綠色螢光筆。#綠色螢光筆專用於「當日退房」，用於預進房會導致顏色判讀混亂。
}

::OP-FO-25-Q06::若當日有外籍旅客預計退房，在房況表上除了標記退房顏色外，還需要做什麼特別註記？ {
=在該房號處註記星號 (*)。#SOP 特別規定若退房為外國人使用之房間，必須在房況表上注記星號。
~在該房號處註記該旅客的國籍名稱。#SOP 簡化標記為「星號」，並未要求寫出具體國籍。
~在該房號處用紅筆圈起來。#紅筆或紅色螢光筆另有他用，SOP 對於外籍退房的規範是使用星號。
~在該房號備註欄寫下「Foreigner」。#SOP 統一使用星號標記，以維持表格簡潔並符合內部標準代碼。
}

::OP-FO-25-Q07::當你在排房時發現 10 樓的 CT 房型旅客備註需要「併床」，你應該如何處理排房表？ {
=直接在排房表上將房型狀態更改為「併」。#SOP 提到 CT 房型為兩小床，若有併床需求，應直接在排房表上更改為「併」。
~在房況表的備註欄寫上「需要併床」，由房務現場判斷。#SOP 要求直接在排房表上進行狀態更改，而不僅僅是寫在備註欄。
~將該訂單更改為標準大床房 (DBL) 以節省作業時間。#任意更改旅客預訂的房種（CT 轉 DBL）可能違反合約或影響排房規劃。
~不需在表單註記，只需口頭交代交班人員即可。#口頭交接容易遺漏，SOP 規定需在排房表上進行正式更改。
}

::OP-FO-25-Q08::若一位旅客在官網預訂了住宿並加購高鐵聯票，在製作 R 卡時，備註欄位的金額應如何填寫？ {
=只寫出總房價，不含高鐵票的金額。#根據補充說明 2，高鐵聯票訂單的備註金額僅需寫出總房價，避免與票價總計混淆。
~寫出房價加上高鐵票價的總金額。#這會導致帳務核對時無法正確區分房費與代收票價，不符合 SOP 規範。
~不需要寫任何金額，只需備註「已付聯票」。#SOP 要求「金額只需寫出總房價」，而非完全不寫。
~將房價與高鐵票價拆開，分別寫在備註欄。#SOP 的簡化規範是僅列出「總房價」，無需在備註欄詳細拆解票價金額。
}

::OP-FO-25-Q09::在填寫「房況表」時，如果某個房間的住客需要使用「床圍」，你應該將此資訊填寫在何處？ {
=填寫在房號的備註空格中，並註明備品名與數量。#SOP 規範應在房號的備註空格中寫上當日入住房客需要的備品名及數量（如：床圍*1）。
~填寫在 R 卡的右下角空白處。#R 卡主要是給旅客確認資料與備註付款狀態，備品調度應記錄在房況表上。
~填寫在德安系統的「訂單備註」即可，不需寫在紙本表單。#雖然系統需記錄，但房況表是現場作業與房務溝通的核心工具，必須依 SOP 填寫。
~記錄在交班簿上，不需影響房況表的格式。#SOP 已明確規定備品需求應寫在房號旁的備註格中，以便房務作業時一目了然。
}

::OP-FO-25-Q10::關於 602 號房的房況作業，下列哪項描述符合 SOP 的特殊執行說明？ {
=它是特殊房型，需確認當日要做 SF（單床）或 DD（雙床）並填入表內。#SOP 第 9 點明確指出 602 為特殊房型，需在查房狀況表上確認並填寫 SF 或 DD。
~它是避難設備房，除非其他房間滿房，否則不排入 602。#避難設備房是 202, 305, 606, 807，602 是屬於「特殊房型」需確認配置，而非避難房。
~它是角落房，排房時應盡量跳過此房號。#SOP 雖說避開角落房，但對 602 的具體指令是確認 SF/DD，而非全面避免安排。
~它是唯一需要用紅鉛筆圈選的預進房號。#SOP 規定預進房號統一用「鉛筆」圈起，並未對 602 規定特殊的顏色標記。
}
