$CATEGORY: OP-FO-20 修改:取消政策執行作業

::OP-FO-20-Q01::有一位透過官網訂房的旅客，預計在 4 天後入住，但剛才來電表示因故需取消訂房。身為櫃檯人員，在執行取消作業前，最優先的操作是什麼？ {
=詢問取消原因並詢問是否願意改期#這能協助飯店了解客群流失原因，並透過延期保留訂單，減少房費損失。
~直接在系統執行取消並告知下次再來#直接取消會忽略轉化訂單的機會，且未依流程紀錄取消原因。
~告知此為免費取消期限，請客人自行至官網處理#雖然可免費取消，但現場人員應主動協助處理並進行必要的溝通詢問。
~確認信用卡號碼以便進行退款作業#此情境為入住前三日以上，若尚未扣款則無需退款流程，應先確認取消意向與原因。
}

::OP-FO-20-Q02::旅客在入住前兩天要求取消訂房，當您告知需酌收 50% 房費時，旅客反應非常強烈且情緒激動。此時最妥當的處理方式為何？ {
~堅持觀光局規定，告知無法優待#僵化的態度容易造成客訴，忽略了異常處理中的回報機制。
=確實記錄客人的情況與反應，並呈報主管協調退款事宜#遇到較大反應時，應紀錄詳細資訊後尋求主管授權處理，以平衡規定與顧客關係。
~為了平息客人怒火，直接答應免收手續費#第一線人員無權私自更改取消政策，應經由報備程序處理。
~建議客人改為入住當日再取消，規避 50% 費用#入住當日取消將收取 100% 費用，此建議會造成客人更大的損失。
}

::OP-FO-20-Q03::一位官網訂房的旅客決定將訂單延期，但他目前還無法確定具體的入住日期。根據作業規範，您應該如何處理這筆訂單？ {
~告知無法保留不確定的日期，請先辦理退費#這會損失已入帳的訂金，且不符合延期保留三個月的政策。
~請客人確定日期後再打來，先不做任何動作#未在系統紀錄或調整，會導致訂單狀態不明確且難以追蹤保留期限。
=先幫客戶保留到三個月期限的最後一天，並告知客人此期限#對於不確定日期者，設定最後期限能確保飯店權益，同時給予顧客彈性空間。
~無限期保留訂金，直到客人主動聯絡為止#長期未結的訂單會造成帳務管理困擾，且依規定逾期三個月後應沒收訂金。
}

::OP-FO-20-Q04::在處理「官網訂房」的退款申請單（UOF系統）時，下列哪一項是必須上傳的附件圖片？ {
~客人的身分證件影本#取消政策流程中並未要求留存客人身分證件圖片。
=德安訂金狀態明細表與 WEB HOTEL 訂單截圖#這些截圖是證明退款金額與原始訂單正確性的重要審核依據。
~旅客手寫的取消申請書#作業流程主要依據系統內的單據與截圖，不包含手寫申請書。
~當初刷卡的信用卡正反面影本#基於資安原則，禁止且不需要留存信用卡正反面圖片。
}

::OP-FO-20-Q05::一位在網路平台（如 Booking.com）訂房的旅客，打電話要求飯店幫他取消當日的訂單。您應如何回應？ {
=請客人自行聯繫網路平台處理，並告知飯店端無法操作#網路平台訂單受平台合約約束，必須由平台端發起取消流程，飯店無法代勞。
~直接在德安系統點選取消單筆訂單#手動取消德安訂單若未經平台確認，會造成平台方仍視為有效訂單而產生佣金或爭議。
~告知客人今天取消可退一半費用#各平台取消政策不同，且當日取消通常不退款，不應給予錯誤的退款承諾。
~請客人提供銀行帳號，飯店直接匯款退費#平台代收的款項應透過平台退款，飯店直接退款會導致帳務錯亂。
}

::OP-FO-20-Q06::針對「無特別規範」的網路平台訂單，若旅客在入住前 5 天提出取消，依照 SOP 應如何執行？ {
~符合免費取消規範，告知客人直接在平台按取消#網路平台若無特別規範，入住七天內取消通常是不符合免費取消政策的。
=告知此情況不符合退款政策，平台會收取全額款項#依 SOP，無特別規範之平台訂單入住七天內為不可退款期，需明確告知權益。
~告知客人可保留金額延期三個月內入住#平台訂單通常不適用飯店自有的延期政策，應以平台的規範為主。
~請客人先聯絡主管爭取免扣款#應先明確告知政策，若客人有疑慮應請其聯繫平台客服，而非直接推給主管。
}

::OP-FO-20-Q07::在完成取消或退款作業後，關於德安系統的「歷史備註」，下列哪項操作是正確的？ {
=在訂單 Profile 備註輸入退款日期、金額及申請人名稱#詳盡的歷史備註能確保後續查詢時，清楚了解該筆款項的去向與經辦人。
~只要在 UOF 系統填寫即可，德安不需紀錄#德安為前台主要管理系統，必須保留完整的訂單變動軌跡。
~直接刪除該筆訂單以清出房位#訂單應執行「取消」而非刪除，否則會失去所有歷史紀錄與追蹤資訊。
~在房型名稱後面加上「已退款」字樣#這不是標準的紀錄位置，應在 Profile 的備註欄中紀錄詳細資訊。
}

::OP-FO-20-Q08::若旅客當初訂房是使用「信用卡預先授權」而未實際扣款，現在要辦理免費取消，櫃檯應如何處理？ {
~請客人提供銀行帳號，以便退還預授權金額#預授權僅是佔用額度而非實際轉帳，無須且無法透過匯款方式退還。
=告知客人飯店會進行取消預授權，额度將會返還#取消預授權會解除信用額度佔用，不會產生實際扣款帳單，這是正確的處理說明。
~必須先將預授權請款，再執行退費流程#這會產生額外的手續費且作業冗長，直接取消預授權才是最有效率的做法。
~告知客人需等三個月後額度才會自動恢復#飯店端手動取消後通常數日內即可恢復，不應給予錯誤的長時等待資訊。
}
