$CATEGORY: OP-FO-18 OTA 平台基礎

::OP-FO-18-Q01::當櫃檯夥伴在核對 Booking.com 訂單時，發現系統顯示「預先授權失敗」，根據標準作業流程，第一時間應如何處理？ {
~直接撥打電話給旅客要求匯款，若不匯款則逕行取消訂單。#SOP 規定 Booking.com 授權失敗應優先於後台留言告知，並非第一時間要求改為匯款或取消。
=至 Booking.com 平台後台留言告知旅客需更換信用卡，並同步標註信用卡無效。#這是針對 Booking.com 授權失敗的標準處理動作，確保有透過系統紀錄並通知旅客更新資訊。
~先不做處理，等旅客入住當天現場刷卡即可，以免影響旅客心情。#忽略授權失敗會增加訂單 नो-शो (No-show) 無法扣款的風險，違反作業安全原則。
~向飯店經理回報系統故障，請資訊單位檢查信用卡授權機台。#授權失敗通常是卡片端問題，應先依 SOP 在平台端執行標註與通知，而非判斷為系統故障。
}

::OP-FO-18-Q02::夥伴正在處理「電話訂房」與「Booking.com」的訂單（項目 1-1），若發現旅客超過三日仍未匯款或預先授權持續失敗，應採取什麼行動？ {
=主動聯繫並詢問顧客狀況。#根據作業說明，三日內未匯款或授權失敗屬於異常狀況，需主動聯繫顧客確認入住意願或支付方式。
~在系統備註「此顧客信用不良」，並通知房務部不用準備該房間。#未完成匯款不代表信用不良，且在未聯繫確認前不應擅自停止房務準備。
~等到入住前七日聯繫時再一併詢問付款狀況，節省通話時間。#SOP 明確指出針對「三日內」未完成者需聯繫，與「入住前七日」的例行確認是分開的流程。
~不需聯繫，系統會自動在入住前一日發送簡訊提醒付款。#前一日的簡訊是「提醒入住」，而支付異常則需要在更早期（三日內）進行人工聯繫確認。
}

::OP-FO-18-Q03::在與「官網訂單」的旅客進行聯繫確認時，哪一項資訊是特別需要與顧客核對的重點？ {
~詢問旅客是否需要代訂火車票。#此項目不在 SOP 的官方網站訂單確認規範中。
=與顧客確認專案內容。#SOP 特別強調官網訂單需與顧客確認專案內容，以避免現場對優惠或配套產生誤解。
~與顧客確認早餐內容，即使專案沒含也要問。#雖然確認早餐是 OTA 訂單的重點，但官網訂單最核心的核對項是「專案內容」。
~僅需確認預抵時間即可，官網訂單不需確認專案。#官網常有各種不同優惠專案，忽略確認可能導致入住時的爭議。
}

::OP-FO-18-Q04::某位 Agoda 的旅客即將在 7 天後入住，但訂單中並未留下聯絡電話。根據流程，你應該如何執行「入住前七日聯繫」？ {
~因為沒有電話，直接略過聯繫步驟，等旅客現場出現再問。#略過聯繫會導致無法掌握停車需求，違反作業流程 1-2 的規範。
=在 Agoda 平台後台留訊息詢問需求。#SOP 規定若訂單無電話，應在平台後台留言進行確認，確保服務不中斷。
~查詢旅客的電子郵件並發送個人信件告知飯店規約。#流程優先建議使用後台留言，而非繞過平台使用個人郵件，且重點應在預抵時間與需求。
~直接打給 Agoda 客服人員，要求客服人員馬上提供旅客的私人電話。#只有在「特殊事項需直接聯繫」且無電話時才致電平台客服，一般例行詢問應優先使用留言功能。
}

::OP-FO-18-Q05::入住前一日的提醒服務中，若發現該筆訂單（如 Expedia）完全沒有旅客電話，該如何完成此項作業？ {
=不發簡訊，改在後台留言提醒入住。#作業 1-2 提到入住前一日寄提醒簡訊，若無電話則在後台留言，確保旅客收到資訊。
~改打給訂房平台客服，請客服逐一唸出旅客的聯絡地址。#這不符合提醒入住的效率原則，且地址對前一日提醒入住的實質幫助不大。
~將該筆訂單標註為異常，交由夜班主管處理。#無電話屬於常見狀況，應依留言流程處理，而非將其列為異常交辦事項。
~略過該旅客，僅針對有電話的旅客發送訊息。#所有旅客都應獲得提醒，無電話時需尋求替代方案（後台留言）而非放棄。
}

::OP-FO-18-Q06::針對 Booking.com、Agoda 或 Expedia 的訂單，在與旅客核對資訊時，「早餐」的確認標準為何？ {
~只有旅客主動問起時才需要確認。#SOP 要求需主動與顧客確認，並非被動等待詢問。
~不論訂單內容為何，一律告知飯店不提供早餐。#這與事實不符，應是確認「有無含」早餐，而非統一否定。
=需主動與顧客確認訂單內容是否有包含早餐。#這是 OTA 訂單確認流程中的明文重點，用以對齊飯店與旅客之間的權益認知。
~統一在入住當天由櫃檯人員判斷，不需要提前聯繫確認。#提前確認可以減少現場 Check-in 的時間，並降低旅客因期待落差產生的不滿。
}

::OP-FO-18-Q07::若旅客有特殊需求或緊急事項（例如更動人數）必須立即聯繫，但訂單中無電話，且在後台留言後旅客遲遲未回覆，此時應如何處理？ {
~放棄聯繫，等旅客到店後再處理糾紛。#這會導致現場混亂，不符合專業服務精神。
~在 Google 搜尋旅客姓名，嘗試尋找對方的聯絡方式。#這可能涉及個人資料安全疑慮，且並非 SOP 規定的專業路徑。
=致電平台客服人員，請客服人員代為聯繫旅客。#這是 SOP 在針對旅客有特殊事項需直接聯繫，卻無法透過電話聯繫時的最終標準指引。
~直接將訂單暫時保留，不安排任何房號。#未經聯繫確認就隨意變更訂單狀態是不正確的，應先尋求平台客服協助對接。
}
