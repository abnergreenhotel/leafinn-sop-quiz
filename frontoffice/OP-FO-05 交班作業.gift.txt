$CATEGORY: OP-FO-05 交班作業

::OP-FO-05-Q01::午班夥伴在執行德安系統接班作業時，於「開關班」介面應輸入哪一個班別代碼？ {
~1#代碼 1 專用於早班時段的系統登錄與結帳作業。
=2#根據飯店前台系統規定，午班對應的班別代碼為 2，用以區分不同時段的帳務。
~3#代碼 3 是保留給大夜班（夜班）夥伴進行系統操作時使用的。
~0#系統班別代碼僅限於 1、2、3，並無 0 的設定。
}

::OP-FO-05-Q02::當客人持「國民旅遊卡」要求辦理住房結帳時，櫃檯夥伴應選擇哪一台刷卡機進行操作？ {
~彰化銀行卡機#彰化銀行機台主要處理一般信用卡（Visa/Mastercard/JCB）與行動支付。
=玉山銀行卡機#玉山銀行機台設定為特定用途機台，專門用於處理國民旅遊卡的交易。
~飯店專用感應讀卡機#一般感應讀卡機若未經特定銀行授權，無法正確辨識國民旅遊卡的消費屬性。
~任一機台皆可，只要金額正確#不同銀行的機台針對特定卡別（如國旅卡）有不同的核銷設定，不可混用。
}

::OP-FO-05-Q03::刷卡機執行關班結帳後，產出的三張單據應依序釘在一起，正確的擺放順序為何？ {
~明細表、總額報表、結帳報表#明細表通常包含單筆資料，應放在最後方作為輔助核對，而非第一張。
~總額報表、結帳報表、明細表#總額報表雖重要，但最優先呈現的應是顯示該機台結帳成功狀態的報表。
=結帳報表、總額報表、明細表#標準流程規定由大項到細項排列，首位為結帳報表，其次為總額彙整，最後才是明細。
~隨機排列，只需包含這三種報表#統一的排列順序有助於稽核人員快速判讀帳務狀態，減少行政錯誤。
}

::OP-FO-05-Q04::在德安系統完成開關班作業（選取班別、按右鍵開班/關班）後，下一步必須執行什麼動作才能確保帳務紀錄正確切換？ {
~直接開始操作訂單#若未重新登入，系統可能仍停留在上一個班別的作業權限或參數中。
=重新登入系統#系統在變更班別後需要重新驗證登入狀態，以啟動新班別的帳務累計。
~列印上一班的日報表#列印報表應在關班前或系統切換完成後執行，並非開班後的首要動作。
~關閉電腦電源#僅需重新登入軟體系統，無須執行硬體重新啟動。
}

::OP-FO-05-Q05::夥伴清點當班零用金時，若金額與作業標準相符，抽屜內應維持多少現金？ {
~$\\$5,000$ 元整#此金額低於標準配置，可能導致次班找零不足。
=$\\$10,000$ 元整#SOP 明確規定零用金需清點並固定維持為一萬元整，以確保營運順暢。
~$\\$12,000$ 元整#超過標準的零用金應作為營收收入包入帳包，不應留在零用金內。
~視當天營收狀況而定#零用金為固定準備金，不應隨每日營收波動，營收應獨立清點。
}

::OP-FO-05-Q06::執行刷卡機關班流程時，為何 SOP 強調「列印單據時，順序不可交換」且需格外謹慎？ {
=因為資料一經清機便會全部清除#一旦執行清機（Settle/結帳），機台內暫存的交易資料會被抹除，若未先印好報表將無法補救。
~因為紙捲長度有限#紙捲長度雖需注意，但並非操作順序不可交換的核心安全性原因。
~因為會導致信用卡公司重複扣款#單據列印順序僅影響報表記錄，不涉及交易後端的扣款授權。
~因為卡機螢幕會鎖死無法操作#卡機不會因為列印順序而鎖死，主要風險在於資料的永久消失。
}

::OP-FO-05-Q07::交班時發現「實收信用卡簽單加總」與「系統交班日報表」金額不符，第一步應如何處理？ {
=重新分類、加總各刷卡簽單並核對#在通報異常前，應先排除人工分類錯誤或漏加特定類別簽單的可能性。
~直接在日報表上手改金額#嚴禁隨意更改系統報表數字，必須找出帳務不符的實際原因。
~將差額由零用金中撥補#零用金不得用於填補信用卡帳務差異，且應維持一萬元整。
~先不管差異，直接進行下班開班#帳務不符若不當班解決，會導致後續稽核困難且責任歸屬不明。
}

::OP-FO-05-Q08::填寫「每日交班稽核表」時，除了確認現金與刷卡單是否相符外，還必須檢查哪一項內容？ {
~當天剩餘的紙捲數量#耗材檢查雖重要，但非「交班稽核表」中列出的帳務核對關鍵項目。
=有無作廢發票#作廢發票涉及稅務與財務審核，必須在交班時與實收帳務一併確認並記錄。
~下一班夥伴的密碼#系統密碼屬個人隱私與資安範疇，嚴禁在交班表上傳閱或填寫。
~客人的聯絡電話#客人資訊紀錄於前台系統內，不需特別在交班稽核表中逐一核對。
}
