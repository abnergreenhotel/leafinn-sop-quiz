$CATEGORY: OP-FO-08 收款入帳作業

::OP-FO-08-Q01::當顧客完成訂房後，根據標準作業流程，應告知顧客最晚需在幾天內完成匯款，以免影響訂房狀態？ {
~訂房完成後當天#要求當天付款過於嚴苛，不符合流程中給予顧客的緩衝時間。
~訂房完成後 2 天內#這比標準作業指導書中規定的期限少了一天。
=訂房完成後 3 天內#根據匯款流程，需明確告知顧客在訂房完成後的 3 天內完成付款作業。
~入住前 3 天#流程規定是以「訂房完成後」作為起算點，而非入住日期。
~入住當天現場付款即可#匯款流程是為了預先收受訂金或房款，並非現場結帳流程。
}

::OP-FO-08-Q02::在執行「匯款查帳」作業時，進入玉山銀行系統後，應點選哪一個帳號進行交易明細查詢？ {
~第一個帳號 (末五碼 12345)#系統中有多個帳號，必須選擇正確的收款專戶以免查不到帳。
=第二個帳號 (末五碼 26358)#作業規範明確指出應點選第二個帳號（末五碼為 26358）進行後續的查帳與核對。
~最後一個帳號 (末五碼 99999)#這是錯誤的帳號代碼，不符合標準查帳步驟。
~隨機點選一個帳號查詢#查帳必須精確，隨機選取會導致工作效率低下且可能誤判款項。
~玉山銀行的總帳戶#銀行網頁介面需點選特定的分戶帳號，而非直接查看總帳。
}

::OP-FO-08-Q03::若顧客提供的是「中信簽帳金融卡 (例如 4477-5787)」且刷卡機無法執行預先授權時，下列處置方式何者正確？ {
~強行輸入卡號直到成功為止#簽帳金融卡若不支援預先授權，多次嘗試也無法解決系統限制。
~請客人聯繫銀行開通授權功能#雖然是一個建議，但流程中首選的是更換支付工具以確保即時性。
=請客人更換信用卡，若無信用卡則改採匯款方式#當簽帳金融卡無法授權時，應優先引導顧客更換一般信用卡或轉為匯款。
~直接取消該筆訂單#在未與客人溝通更換支付方式前直接取消訂單會引發客訴。
~告知客人系統維修中，請稍後再試#這屬於欺騙顧客，並非處理金融卡無法授權的標準程序。
}

::OP-FO-08-Q04::在執行 BOOKING 信用卡授權作業時，若授權失敗，下列哪一項不是標準流程中規定的後續處理步驟？ {
~於 BK 訂單明細將信用卡標示無效#這是標註通路系統付款狀態的重要步驟。
~傳送罐頭簡訊給客人#需透過簡訊即時通知顧客卡片問題，以便顧客更新資訊。
~於德安訂單備註說明失敗原因#在內部飯店系統中記錄失敗原因，有助於櫃檯交接。
~填入 NOTION 中授權失敗的表單內#這是為了後續追蹤與管理未成功收款訂單的必要步驟。
=撥打電話至銀行要求人工授權#流程中並未要求執行人工授權，而是轉由通路標示無效並通知顧客。
}

::OP-FO-08-Q05::辦理「授權完成」作業時，除了輸入金額與感應卡片外，刷卡機還需要依序輸入哪些資訊？ {
~身分證字號、授權碼、卡片效期#授權完成不需要輸入客人的身分證字號。
=授權碼、序號、系統追蹤號#根據流程，補登交易需輸入原預先授權的授權碼、序號及系統追蹤號以進行對帳。
~顧客姓名、電話、訂單編號#刷卡機端的補登操作主要輸入的是與銀行交易相關的代碼。
~房間號碼、入住日期、退房日期#這些資訊雖然重要，但並非刷卡機執行授權完成時要求的必填欄位。
~信用卡末四碼、CVV 碼、出生年月日#補登作業不需要 CVV 碼或出生年月日。
}

::OP-FO-08-Q06::顧客因故更改付款方式，需執行「授權取消」作業，服務人員應告知顧客退還額度的作業時間約為多久？ {
~當天立即退還#銀行端處理刷退或取消授權通常無法即時反映在客人的額度上。
~1 到 3 個工作天#這通常是國內銀行轉帳的時間，非信用卡額度釋放的標準時間。
=7 到 14 個工作天#話術中明確提到，取消授權的申請與額度退還需依照銀行作業，約需 7 到 14 個工作天。
~一個月後的結帳日#這時間過長，一般銀行作業不需等到下個月結帳日。
~視客人信用卡的發卡行而定，無法預估#雖然確實依銀行而定，但作業標準話術中仍有提供一個參考區間。
}

::OP-FO-08-Q07::在德安系統執行「入訂金流程」時，若顧客入住多天但表示「只需要一張總發票」，發票開立方式應如何選擇？ {
~選「不開」#無論如何都需要開立發票，不能選擇不開。
~選「後開」#「後開」通常用於一天一張發票的情況。
=選「已開」#根據補充說明 *6-2，若入住多天只需一張總發票，系統應選擇「已開」。
~由系統自動判定#系統無法得知客人的發票偏好，必須由操作人員手動選取。
~點選「暫存」待退房再開#流程中並無暫存選項，需在入訂金時決定發票開立原則。
}

::OP-FO-08-Q08::關於「開立公帳號流程」，應在哪個管理功能介面中點選房號按右鍵進行設定？ {
~出納管理 > 住客帳維護#住客帳維護是用於結帳，而非設定房間的公帳號屬性。
=接待管理 > 房間管理#標準流程規定在「接待管理」的「房間管理」介面下設定公帳號。
~訂房系統 > 查詢訂單#訂房系統主要處理預約資訊，房間的細部狀態管理在房間管理模組。
~系統設定 > 參數管理#這屬於高階權限設定，非日常作業的公帳號開立流程。
~報表管理 > 房務報表#報表管理僅供查看與輸出資料，無法進行公帳號的動態設定。
}

::OP-FO-08-Q09::在辦理退房結帳時，若要沖抵先前已收取的「訂金」，德安系統的「付款方式代碼」應選擇哪一個？ {
~01 現金結帳#現金結帳是指現場收現，而非扣抵已付訂金。
~02 信用卡結帳#這用於結清尾款時刷卡，並非沖抵訂金。
=00 沖客房訂金#結帳流程第 9 步明確指出，有預收收款時應選擇「00 沖客房訂金」進行帳務沖抵。
~99 客戶簽帳#客戶簽帳多用於訂房網站或掛帳客戶，不適用於一般散客訂金沖轉。
~88 住宿券結帳#住宿券有其專用的代號，與訂金沖銷不同。
}

::OP-FO-08-Q10::當旅客在入住期間產生了「其他費用」（如備品購買或付費設施），應在德安系統執行哪一項操作來將費用計入房帳？ {
~點選「訂單備註」寫下費用內容#僅在備註欄寫下不會產生實際帳務，會導致漏收款項。
=點選「旅客帳處理」中的「入帳」#根據其他費用結帳流程，應點選「旅客帳處理」並執行「入帳」來輸入消費項與金額。
~直接在「住客帳維護」修改總金額#總金額應由各項消費累計而成，不應隨意手動更改總數。
~進入「訂房系統」修改房費#其他費用不等於房費，應分開項目入帳以利報表分析。
~點選「預估款」直接結帳#「預估款」通常用於檢視現有帳務，入帳需先點選消費項。
}
