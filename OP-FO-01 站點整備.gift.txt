$CATEGORY: OP-FO-01 站點整備

::OP-FO-01-Q01::在執行「櫃台環境檢查」時，除了確認桌面無雜物外，下列哪一項也是該項目的執行重點？ {
~確認當日所有客房的維修狀態#維修狀態屬於作業日誌與當日重點掌握的範疇，而非環境檢查的基礎內容。
~清點房卡與停車券的剩餘數量#備品清點屬於獨立的第二項作業，與環境整潔的初步巡視不同。
=確保燈光、音樂與空調符合營運需求#環境檢查不僅包含視覺的整潔，也包含氛圍控制與硬體設施（如空調、門口狀態）的營運設定。
~測試 PMS 系統與刷卡機功能#設備功能測試是環境檢查之後的專門程序，旨在確保軟硬體連線正常。
~回報當班可正式接待客人的狀態#回報正式接待是開站流程的最後一個步驟，而非環境檢查的一部分。
}

::OP-FO-01-Q02::關於備品清點與補充的原則，下列敘述何者正確？ {
~為避免浪費，應等到顧客提出需求時再行補充#標準規範要求備品不足需當班即時補充，避免在顧客使用時才處理，以維護服務效率。
~清點項目僅限房卡與發票紙，其他文具由行政人員處理#櫃台夥伴需清點包含租借表單、寄物牌、地圖與文具在內的所有常用備品。
~備品清點僅需在夜間時段進行，開站時不需執行#開站整備流程明確要求在上班後需進行備品與文件清點，以應對當日營運。
=若發現備品不足，應先補足至規定的安全庫存量#維持安全庫存是備品管理的關鍵，能確保在突發高需求量時仍有足夠資源服務顧客。
~文件清點只需確認是否有發票紙，敏感文件不在此限#敏感文件的處理主要在環境檢查中確保不外露，且所有常用表單皆在清點範疇內。
}

::OP-FO-01-Q03::若在「設備功能確認」階段發現刷卡機連線異常，應如何處理最符合 SOP？ {
=先行測試基本排除並立即回報主管#發現異常時需具備初步排除能力並同步通報，以避免問題拖延至營運高峰期。
~先處理顧客退房，等空閒時再行報修#規範中強調不可等到尖峰時段再處理異常，以免影響現場服務順暢度。
~不需理會，直接交由資訊人員定期維護即可#站點整備要求開站前需確認所有設備正常，不應將現有故障留待後續維護。
~僅需重啟 PMS 系統，因為刷卡機異常通常是系統問題#排除動作應針對異常設備本身（刷卡機），且需有測試與回報的完整動作。
~自行拆解設備檢查線路，不需麻煩主管#規範要求的是基本排除與回報，過度的自行拆修可能導致設備損壞。
}

::OP-FO-01-Q04::在「交班紀錄簿閱讀」項目中，若夥伴尚未讀完前一班的紀錄，應如何行動？ {
~可以先幫現場排隊的客人辦理入住，事後再補讀#規範明確規定未讀交班紀錄者不得直接接手櫃台，以避免資訊斷層導致服務錯誤。
~請前一班夥伴口頭快速帶過即可，不需閱讀文字#閱讀文字紀錄包含未完成案件與特殊細節，口頭摘要容易遺漏重要帳務或客訴提醒。
=必須詳讀完畢後，方可正式接手櫃台事務#確保對前一班遺留案件、VIP 與特殊房況有完整掌握，是提供延續性服務的基礎。
~只需閱讀關於「客訴」的部分，其餘維修資訊可略過#閱讀範疇包含交接事項、未完成案件、房況、帳務與停車等多個層面。
~若前一班表示「沒事」，即可不閱讀直接簽名#即使前班表示無事，接班者仍有責任檢視完整紀錄以確認無漏掉的細節。
}

::OP-FO-01-Q05::夥伴在詢問前班同事：「請問今天有什麼重要事項需要交接的嗎？」時，其核心目的為何？ {
~為了練習標準禮儀話術#話術只是溝通工具，其核心目的在於確保服務資訊的無縫接軌。
=確認是否有需要提前安排的顧客需求或特殊活動#透過主動詢問與紀錄閱讀，能掌握當日重點並預先為顧客需求做準備。
~核對當日出勤人數是否正確#交接的核心在於業務執行細節與顧客相關事務，而非行政考勤。
~確認前班夥伴是否有準時下班#這句話術專注於工作內容的傳承，並非監督同事的差勤狀態。
~僅為了確認停車場是否還有空位#停車雖然是交接內容之一，但並非該詢問動作的唯一核心目的。
}

::OP-FO-01-Q06::關於「作業日誌與當日重點掌握」的注意事項，下列哪一項做法不正確？ {
~整合判讀當日重點與交班事項#這是標準要求，必須將不同來源的資訊整合分析而非單看單一紀錄。
=僅閱讀當日退房名單，忽略團體與特殊活動資訊#作業日誌要求全面掌握住房、退房、團體、活動與維修等多項重點，不可偏廢。
~確認是否有須提前安排的顧客需求#這是掌握重點後的關鍵應用，屬於正確的服務準備流程。
~檢視主管在日誌中的提醒事項#主管提醒通常涉及營運調整或重要方針，是每日必讀項目。
~將重要案件再次記錄於個人工作清單中#這有助於確保在忙碌的執行過程中不會遺漏關鍵的交辦任務。
}

::OP-FO-01-Q07::執行「開站完成確認」的最後一個動作應為何？ {
~立即開始整理前一日的帳務資料#帳務處理並非開站確認的最後動作，通常是交接或營運中的專門任務。
=回報當班可正式接待#完成所有檢查項目的最後一步是正式宣告狀態，讓團隊知道櫃台已準備就緒。
~關閉所有的走廊燈光以節省電源#環境檢查已要求燈光需符合營運需求，隨意關燈可能違反服務標準。
~重新清點一次所有的備品庫存#備品清點在第二步驟已完成，最後步驟不需要重複執行同一動作。
~檢查個人手機訊息是否有私事待辦#開站程序中應確保桌面無私人物品，處理私事不屬於標準作業流程。
}

::OP-FO-01-Q08::若你在值勤期間發現大廳門口有客人的垃圾，而你已經在早晨完成「開站整備」，你該如何處理？ {
~不需理會，因為環境檢查在開站時已通過#規範強調開站完成不代表後續可忽略巡檢，需持續維持環境整潔。
~等待下一班交接時再請接班夥伴處理#這違反了當班即時維護的原則，會導致服務品質下降。
=持續補強並維持環境整潔，即時清理#值勤中若發現環境異常，應秉持持續維持的精神進行補強，確保隨時處於最佳狀態。
~記錄在交班簿上即可，不需動手清理#現場問題應立即解決，僅做記錄而未改善會影響當下的顧客體驗。
~請清潔人員在下一次固定清掃時間再來處理#櫃台夥伴負有巡視與維持環境整潔的責任，不可完全依賴固定清掃。
}

::OP-FO-01-Q09::在「備品與文件清點」中，哪一項物品的補給不需要達到「安全庫存」？ {
~美食護照#美食護照屬於常用備品，必須維持足夠數量。
~發票紙#發票紙是營運必需品，若不足會導致交易中斷，必須維持庫存。
=所有列出的常用備品皆需補足至安全庫存#SOP 明確指出包含房卡、表單、文具等在內的所有常用備品，若不足皆需補至安全庫存。
~僅有房卡需要維持庫存，文具不需要#文具亦被歸類在常用備品中，同樣需要清點與補充。
~顧客寄放的行李#寄放行李是服務項目而非「清點補給」的耗材備品，題目語境不符。
}

::OP-FO-01-Q10::針對「敏感文件外露」的檢查，主要是在防範下列哪種風險？ {
~防止文件被空調吹散#雖然也是問題，但環境檢查強調的是「敏感性」，與資訊安全或隱私較相關。
=確保顧客資訊隱私與營運機密不被窺視#櫃台桌面是顧客可見區域，不應放置包含個資或敏感資訊的表單，以符合專業標準。
~減少櫃台人員尋找文件的時間#這是組織管理的益處，但檢查「敏感文件」的主因是安全防護。
~確保文件有足夠的空白處可以寫字#這屬於文件的可用性，而非環境檢查中提到的「外露」問題。
~避免文件被陽光曬到褪色#此說法與酒店營運環境的實務操作無關。
}
