$CATEGORY: OP-OB-04 隱私資安

::OP-OB-04-Q01::根據「個人資料保密宣導」，下列哪一組資訊皆屬於必須嚴格保密的住客機密資料？ {
~僅限住客姓名與聯絡電話。#除了基本聯絡資訊，身分證件、房號及付款紀錄同樣屬於保密範疇。
~僅限身分證件字號與付款資料。#此選項忽略了住客的住宿紀錄與房號等關鍵營運資訊。
=住客姓名、電話、身分證件、房號、付款資料與住宿紀錄。#規範中明確界定所有與住客身分、消費及留宿相關的細節均為機密。
~僅限數位格式的資料，紙本名單不在此限。#無論是數位或紙本格式，只要涉及住客隱私皆須受到保護。
~僅限住客主動要求保密的特殊需求內容。#保密是飯店作業的基本原則，並非僅針對住客要求的特定項目。
}

::OP-OB-04-Q02::當有熟客或是住客的親友致電詢問其入住房號時，身為夥伴應如何反應？ {
~若是熟客，為了維持關係應直接告知房號。#規範中明確指出不可因熟客身分而透露住客資訊。
~確認對方身分為親友後，即可告知相關資訊。#親友身分並非解除保密原則的例外條件，仍須遵守隱私保護原則。
=婉拒提供資訊，並堅持遵守隱私保護原則。#對於第三方的查詢，無論身分，皆須嚴格遵守隱私保護原則以保障住客。
~只要對方能說出住客全名，即可核對並提供房號。#僅知道全名不足以授權對外透露住客的具體入住動態。
~請對方留下電話，待詢問住客同意後再回撥。#雖然看似周全，但直接告知資訊或確認住客在館內仍需極其謹慎，標準作業首重不對外透露。
}

::OP-OB-04-Q03::關於系統帳號的使用規範，下列敘述何者正確？ {
~為求效率，同一組的夥伴可以共用一組 PMS 帳號。#規範明訂僅開放工作所需權限，且不得共用帳號以確保責任歸屬。
=帳號與密碼僅限本人使用，不可借用或代登。#這是確保個人行為軌跡清晰且符合資安稽核的核心要求。
~預設密碼安全性已足夠，不需刻意更改。#規範要求首次登入後必須立即更改預設密碼。
~只有管理階層的帳號才需要嚴格禁止共用。#所有帳號均依職務申請，不論職級高低皆不可共用。
~若同事忘記密碼，可以先暫時借用自己的帳號協助其操作。#帳號不得借用，遇此情況應請同事依照流程重設密碼。
}

::OP-OB-04-Q04::在密碼管理規範中，下列哪一種行為是符合資安標準的？ {
~為了方便記憶，使用自己的生日作為系統密碼。#密碼應避免使用生日、電話等容易被猜測的個人資訊。
~將密碼寫在便利貼上，貼在螢幕邊框方便交接。#密碼嚴禁記錄在公開可見處或容易被他人發現的地方。
=若懷疑密碼外洩，應立即修改密碼並回報主管。#這是防範風險擴大的即時應變動作，符合安全通報機制。
~將帳號與密碼傳送到單位的 Line 通訊群組備存。#不可將帳密傳送至公開或通訊群組，以免造成二次外洩風險。
~使用與其他私人帳號相同的連號密碼以減少遺忘機率。#應避免使用簡易連號或共通密碼以強化安全性。
}

::OP-OB-04-Q05::當櫃台人員需要暫時離開工作站前往處理其他庶務時，首要動作為何？ {
~直接離開，只要時間不超過五分鐘即可。#離開櫃台前必須先處理畫面，不可將資訊暴露在無人看管的狀態。
~將螢幕電源關閉即可。#僅關閉螢幕並未鎖定系統，他人仍可輕易重啟查看資訊。
=先登出系統或鎖定畫面。#這是防止資料被非授權人員看見或操作的最基本保護措施。
~請現場排隊的住客幫忙留意一下螢幕。#住客並非工作人員，此舉反而會直接將個資暴露給外人。
~將開啟中的住客資料視窗縮小化即可。#縮小視窗並不足以防止有心人士點開查看。
}

::OP-OB-04-Q06::關於紙本報表或住客名單的處理，下列哪一項敘述符合作業指導書的規範？ {
~使用完畢的名單可以暫時放在公共區域的抽屜內。#公共區域不可放置含個資文件，以防外流。
=列印出的名單若不再使用，應立即回收或銷毀。#這是針對實體個資文件「隨用隨清」的安全處理標準。
~可以帶回家研讀以熟悉住客姓名，提升服務品質。#住客名單嚴禁外流或帶離工作場所。
~將過期的住客清單背面當作筆記紙重複利用。#含有個資的文件必須徹底銷毀，不得任意挪作他用以免個資外洩。
~隨意放置在櫃台後方桌面，只要住客看不到即可。#文件不可隨意放置，必須確保不會流向非授權人員。
}

::OP-OB-04-Q07::關於社群媒體的使用規範，下列哪一種行為是被允許的？ {
~在個人 IG 分享一張拍到內部系統畫面但模糊房號的照片。#內部畫面與系統內容均不得公開張貼，無論清晰度為何。
~未經授權，以公司代表身分在網友討論區回覆營運疑問。#未經授權不得代表公司對外發言。
=轉發公司官方已公開的資訊，但個人不發布未經授權的數據。#規範禁止發布未公開資訊、數據或個資，轉發官方已核准內容則是安全的。
~分享與知名 KOL 在館內的合照並備註其房號以表示禮遇。#住客照片與房號皆屬嚴禁公開的資訊，不論住客身分。
~在個人臉書發布今日飯店的精確營運住房數據。#營運數據屬於館內未公開資訊，嚴禁私自發布。
}

::OP-OB-04-Q08::若在值勤時遇到媒體或網友詢問館內未公開的資訊，應如何處理？ {
~直接告知對方自己所知道的訊息，展現熱誠。#未經授權的個人發言可能導致資訊錯誤或商業機密外流。
=委婉告知目前無法回覆，並統一轉由主管或指定窗口處理。#這是面對外部詢問時的標準流程，確保發言內容一致且經過授權。
~向對方表示這些是機密，絕對不能透露。#語氣可能過於生硬，且應依循標準流程引導至正確窗口。
~將對方加入個人通訊軟體私下回覆。#此舉嚴重違反對外發言與資安規範。
~先自行判斷對方的意圖，再決定是否回答部分內容。#夥伴不應自行判斷，應一律轉介至專責窗口。
}

::OP-OB-04-Q09::當發現系統帳號出現異常登入或收到可疑郵件時，下列哪一項行動是錯誤的？ {
~保留異常發生的現況，不再進行不必要操作。#這是正確的做法，有利於後續調查。
=立即刪除可疑紀錄，以免主管擔心。#規範要求不得自行刪除紀錄或私下處理，必須先保留現況。
~立即通知主管與資訊窗口協助處理。#這是正確的通報流程。
~記錄發生時間、系統名稱及異常內容。#這是正確的紀錄步驟，有助於問題排查。
~依據主管指引採取進一步的後續措施。#這是正確的程序，需優先通報再依指示處理。
}

::OP-OB-04-Q10::當夥伴辦理離職或職務調動時，關於系統帳號權限的處置應如何進行？ {
~直接將帳號密碼交給接任的夥伴接續使用。#嚴禁私下提供密碼或共用帳號。
=由主管或資訊窗口負責立即關閉或調整該帳號權限。#職務異動時必須立即停權，以符合帳號管理的完整性。
~將帳號留著，以防未來需要回公司支援時可以登入。#離職或調職必須立即關閉權限，防止帳號成為資安漏洞。
~請接任者自行嘗試舊密碼登入測試。#帳號與權限必須重新申請或依正規流程變更。
~帳號會隨時間自動失效，不需主動通知處理。#規範要求離職、調職或停權時需「立即」處理。
}
