$CATEGORY: OP-FO-11 退房作業流程

::OP-FO-11-Q01::根據退房作業標準流程，旅客最晚應於什麼時間前歸還房卡辦理退房？ {
~上午十點半前#這比規定的退房時間早了一個小時，並非手冊中提到的截止時限。
~上午十一點前#雖然是一般飯店常見的時間，但手冊中明確規定的版本作業時間為十一點半。
=上午十一點半前#根據「版本1」與注意事項，旅客應於十一點半前歸還房卡完成退房流程。
~中午十二點前#這超過了規定的退房時間，可能會產生加收費用的問題。
}

::OP-FO-11-Q02::若旅客遺失一張房卡，應如何處理收費？ {
~無需收費，直接補發#作業流程中明確規定遺失房卡是有成本賠償責任的。
=一張收取一百元#根據執行說明，若有遺失房卡，需依每張一百元的標準收費。
~一張收取兩百元#兩百元是加休一小時的費用，並非房卡的遺失賠償金。
~一張收取三百元#此金額超出了手冊規定的房卡重製成本費用。
}

::OP-FO-11-Q03::當旅客退房時表示有借用的「餐具組」尚未交還，且可能放在房間內，櫃檯人員優先該怎麼做？ {
~直接向旅客收取清潔費用#在尚未確認物品狀態前，不應直接加收費用，應先進行確認。
~請旅客自行回房間拿下來#這會降低顧客滿意度，流程建議應主動詢問並由內部人員確認。
=詢問房務員是否已收到並清洗完畢#針對需要清洗的租借品（如餐具、酒杯），應確認房務端是否已回收處理。
~直接在德安系統點選退房結帳#在租借物品未核銷或確認前完成結帳，可能導致資產損失。
}

::OP-FO-11-Q04::關於加休費用的計算與規定，下列敘述何者正確？ {
~每小時酌收一百元，不限時數#加休每小時的費用應為兩百元，且有最高時數限制。
=每小時酌收兩百元，最多加休兩小時#手冊規定加休一小時收兩百元，上限為兩小時，之後須完成退房作業。
~每小時酌收兩百元，最多加休三小時#雖然每小時單價正確，但上限應為兩小時而非三小時。
~加休不滿一小時不計費#手冊並未提到不計費的寬限期，應以小時為單位進行收費。
}

::OP-FO-11-Q05::若旅客是透過 Agoda (AG) 或 KLOOK 訂房，要求在櫃檯開立房費發票，該如何回應？ {
~直接幫顧客開立發票並加蓋統編章#這會造成稅務上的重複開立，因為平台端已負責相關帳務。
~請顧客改用現金支付才可開立發票#訂房平台的支付方式與現場開立發票的規範不應以此掛鉤。
=告知發票需向原訂房平台或處申請#手冊明確指出 AG, EXP, KLOOK 等平台訂房皆無現場發票，應向原訂房處申請。
~僅能開立住宿證明，不能給收據#雖然不能開立發票，但回應時應清楚引導顧客去正確的管道索取憑證。
}

::OP-FO-11-Q06::在製作「住宿證明」時，關於房費顯示的規定為何？ {
~不論有無發票，皆必須顯示房費金額#房費顯示與否取決於現場是否有開立發票，以避免帳務混淆。
=僅有開立發票者，方可在證明上顯示房費#根據作業說明，開立發票者可顯示房費，無發票者不可顯示房費。
~為保護隱私，住宿證明一律不顯示房費#若旅客需要報帳且現場有開立發票，住宿證明是可以用於核對金額的。
~所有證明皆不需蓋章，直接列印即可#手冊要求在顧客確認資訊無誤後，必須蓋上公司統編章。
}

::OP-FO-11-Q07::使用德安系統處理退房結帳時，找到房號或姓名後，第一步應點選哪一個選單？ {
~房務管理#房務管理主要負責房間狀態，結帳應在接待相關選單下進行。
~報表管理#報表管理用於輸出數據，而非處理個別旅客的即時結帳與入帳。
=旅客帳處理#在房號上按右鍵後，應點選「旅客帳處理」來進行後續的入帳與結帳動作。
~房卡設定#此功能用於讀寫卡片，與系統上的財務帳務處理流程不同。
}

::OP-FO-11-Q08::在「顧客關懷」環節，若遇到旅客反映住宿期間有不滿意的地方（客怨），下列做法何者不符合 SOP？ {
~先關心客人並致歉#這是處理客怨的基本態度，符合流程中「先關心、致歉」的指導。
~視情況贈送小禮物#流程建議在發生客怨時，可視情況送禮以示誠意。
~若無法處理應立即通知主管#當一線人員權限不足或無法平息不滿時，尋求主管協助是標準步驟。
=告知旅客退房時間已到，請其先離開再投訴#此做法完全違反顧客關懷原則，流程強調應立即處理或通知主管。
}

::OP-FO-11-Q09::退房完成後，關於「旅客住宿登記卡」的處理方式，下列何者正確？ {
~直接交還給旅客帶走#登記卡屬於飯店留存資料，且含有個資，不能隨意讓旅客帶走。
=放置於櫃檯下方留存，避免個資外洩#手冊特別提醒個人個資不可外洩，且須放置於櫃檯下方安全處。
~張貼在辦公室公佈欄方便核對#這會導致嚴重的個資外洩問題，違反隱私保護規範。
~立即放入碎紙機銷毀#法律通常規定需留存一段時間備查，不應在退房後立即銷毀。
}

::OP-FO-11-Q10::在送客流程中，若遇到下雨天，櫃檯夥伴應主動提供什麼服務？ {
~幫顧客叫外送#雖然是服務的一種，但非流程中針對雨天情境特別註記的送客行為。
=幫顧客撐傘#作業流程第7點特別標註「雨天幫顧客撐傘」，展現貼心服務。
~販售拋棄式雨衣#流程強調的是主動的服務（撐傘），而非轉向銷售商品。
~請顧客在門口等雨停#這並非積極的送客與關懷表現，應主動協助顧客離場。
}
