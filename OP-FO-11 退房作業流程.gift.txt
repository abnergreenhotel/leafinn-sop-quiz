$CATEGORY: OP-FO-11 退房作業流程

::OP-FO-11-Q01::根據「OP-FO-11 退房作業流程」，旅客應於何時之前歸還房卡辦理退房？ {
~$10\:00$#這個時間通常是房務開始大規模清掃的時間，但非手冊規定的標準退房時限。
~$11\:00$#雖然許多飯店設定於此時間，但本手冊明確規定了更具體的截止時間點。
=$11\:30$#手冊作業指導書的第一項明確規定，旅客應於十一點半前歸還房卡辦理退房。
~$12\:00$#這是一般飯店常見的退房時間，但與本作業規範中的時間點不符。
~$13\:00$#此時間已屬於加休收費的範疇，而非標準退房時間。
}

::OP-FO-11-Q02::當旅客來到櫃檯表示要退房時，夥伴首先應如何確認資料以確保安全與準確性？ {
~直接向旅客索取身分證件進行複印#退房時的首要步驟是確認房號與登記人資訊的相符性，而非重複入手的身分證件複印流程。
~詢問旅客昨日入住的總金額是否正確#金額確認屬於結帳階段的步驟，在初步確認身分時並非首要動作。
=確認房號與登記人姓名是否與德安系統資料相符#確認房號與登記人資料相符是退房初步核對的核心，若不符則需進一步查詢系統房間管理資訊。
~請旅客先行在櫃檯下方留存的登記卡上簽名#登記卡是在辦理入住時使用，退房時應是將其妥善留存並保護個資，而非再次簽名。
~立即致電房務部確認房間是否有物品損壞#此動作雖重要，但在確認旅客身分與房號之前進行會顯得突兀且不符合標準流程順序。
}

::OP-FO-11-Q03::若旅客在辦理退房時表示遺失了一張房卡，根據作業指導書應如何處置？ {
~請旅客留下聯絡電話，待尋獲後再行歸還#這會增加後續追蹤成本，且不符合當下處理損壞或遺失物品的費用收取規定。
=告知旅客需支付工本費 $100$ 元#根據房卡數量確認規則，若有遺失房卡，一張需收費一百元。
~直接在德安系統中註銷該卡，不予收費#雖然系統需註銷，但手冊明確規範了資產遺失的賠償金額。
~要求旅客必須找回房卡否則不能完成退房程序#這會導致客怨，標準流程應是以收費賠償作為遺失的解決方案。
~視旅客住宿天數決定是否減免該筆費用#標準作業流程中並未提到根據天數減免遺失費用的權限。
}

::OP-FO-11-Q04::旅客若租借了「餐具組」但在退房時未主動繳回，櫃檯夥伴應如何處理？ {
~直接依照損壞賠償標準在結帳時扣款#在未確認物品狀態前直接扣款易引發爭議，應先詢問物品所在位置。
=詢問旅客是否放在房間，並連繫房務確認是否已回收清洗#針對需清洗的租借品，應詢問旅客是否留在房內，並與房務端確認回收狀況。
~請旅客親自回房間將餐具拿下來櫃檯繳回#這不符合優質服務的原則，且手冊建議的是詢問物品位置並與房務聯繫。
~若房務尚未清掃，則直接相信旅客已歸還並完成退房#這可能導致資產流失，必須落實與房務端的雙重確認。
~要求旅客在退房後的一小時內送回櫃檯#旅客已辦理退房通常會離開飯店，此要求不切實際。
}

::OP-FO-11-Q05::關於「加休」的費用與規則，下列敘述何者正確？ {
~加休每小時收費 $100$ 元，最多可加休三小時#金額與時數限制皆與手冊規範不符，手冊規定每小時為兩百元。
=加休一小時酌收 $200$ 元，最多加休兩小時#手冊明確規範加休每小時兩百元，且上限為兩小時，超過則需調整作業。
~若旅客有物品損壞，加休費用可予以折抵#損壞賠償與加休費用應分開計算，手冊並無折抵之說。
~加休時間不需告知房務，僅需在系統入帳即可#手冊強調「確認加休時間並且告知房務」，以便房務調整排班清掃進度。
~加休滿三小時可享發票免稅優惠#此說法完全不符合現行稅務規定，且非手冊內容。
}

::OP-FO-11-Q06::若旅客是透過 Agoda (AG) 或 KLOOK 平台訂房並預付房費，退房時要求開立發票，櫃檯應如何回覆？ {
~直接按照德安系統內的金額開立發票給旅客#OTA 訂房的發票通常由平台開立，飯店若重複開立會導致稅務問題。
~請旅客提供電子郵件，飯店隨後補寄電子發票#此做法錯誤，因為飯店端並非該筆款項的發票開立主體。
=告知旅客需向原訂房平台申請發票#針對 AG, EXP, KLOOK 等平台訂房，飯店不開立發票，應引導顧客回原平台申請。
~告知旅客平台訂房一律不提供任何憑證#旅客仍有權索取憑證，只是索取的對象應是訂房平台而非飯店。
~僅能開立「住宿證明」且必須顯示房費金額#無發票者在住宿證明中不可顯示房費，以免造成帳務混亂。
}

::OP-FO-11-Q07::在「德安系統」操作退房結帳時，關於「入帳」的正確操作流程為何？ {
~房間管理 $\\rightarrow$ 選取確定退房 $\\rightarrow$ 點選入帳#入帳必須在確定退房之前完成，否則帳務無法歸屬。
=接待管理 $\\rightarrow$ 房間管理 $\\rightarrow$ 旅客帳處理 $\\rightarrow$ 點選入帳 $\\rightarrow$ 選取消費項目#這是標準的系統操作路徑，先進入帳務處理界面再選擇具體的消費項目入帳。
~飯店前台系統 $\\rightarrow$ 消費項目 $\\rightarrow$ 輸⼊金額 $\\rightarrow$ 點選結帳#缺少了選取特定旅客房號的關鍵步驟，無法確保帳務正確性。
~電子發票列印 $\\rightarrow$ 點選入帳 $\\rightarrow$ 輸入單價與數量#列印發票是最後一步，不應在入帳之前執行。
~點選結帳 $\\rightarrow$ 選擇付款方式 $\\rightarrow$ 點選入帳#結帳是結清現有帳目，應先入帳（增加項目）再行結帳（支付總額）。
}

::OP-FO-11-Q08::當需要開立「住宿證明」給「沒有開立發票（如平台訂房）」的旅客時，下列哪項資訊「不可」顯示？ {
~公司名稱#公司名稱是證明文件的基本要素，應根據需求填寫。
~住宿日期與退房日期#日期是證明日期的核心資訊，必須正確標示。
=房費金額#手冊特別規定，若是無發票者，住宿證明不可顯示房費，避免帳務爭議。
~旅客姓名#姓名為識別住宿人的必要資訊，必須顯示。
~飯店統編章#蓋上統編章是為了確保證明的正式性與法律效力。
}

::OP-FO-11-Q09::退房作業流程中，關於「顧客關懷/意見調查」的執行重點，下列何者正確？ {
~只需詢問「退房完成了嗎？」即可讓客人離開#這缺乏與顧客的互動，無法收集意見或提升服務滿意度。
=若有客怨，應先致歉、關心，視情況送禮，並視需要通知主管#這是處理客訴與提升滿意度的標準應對程序，強調先處理心情再處理事情。
~若旅客反應不滿意，應立即在系統中將其標記為黑名單#不滿意應透過溝通解決，而非消極處理，手冊並無此要求。
~意見調查僅需針對入住超過三天的旅客執行#手冊並未限制天數，所有退房旅客皆是意見收集的對象。
~僅在旅客主動提及問題時，才需進行關懷問候#夥伴應主動詢問「昨日入住上還滿意嗎？」，這屬於標準話術的一部分。
}

::OP-FO-11-Q10::在「送客」階段，若遇到雨天，標準作業程序規定夥伴應提供什麼服務？ {
~主動幫顧客叫外送食物#送客時顧客即將離開，此服務與當下情境不符。
~提醒顧客雨天路滑，請自行保重#僅有口頭提醒不足以展現積極服務，應有實際行動。
=幫顧客撐傘並執行目送禮#手冊特別備註「雨天幫顧客撐傘」，這是細膩服務的具體表現。
~要求顧客在雨停後再離開以確保行李乾燥#顧客通常有既定行程，應主動提供協助而非限制其離開時間。
~僅需在門口揮手致意，不必走出大門#這不符合「目送離開」與「拿行李」的積極服務精神。
}
