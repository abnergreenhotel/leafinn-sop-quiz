=$\\$10,000$ 元#作業流程明確規定清點零用金應為一萬元整，作為次班作業的基礎金額。
~$\\$15,000$ 元#此數額超過了指導書規定的標準零用金金額。
~當班現金收入總額#現金收入包與零用金應分開清點，零用金是固定留存於櫃檯的預備金。
~視當日住房率決定金額#零用金數額是固定的標準作業程序，不應隨住房率波動而改變。
}

::OP-FO-05-Q02::當櫃檯人員需要為持「國民旅遊卡」的旅客辦理結帳時，根據手冊規定，應優先使用哪一台信用卡機？ {
~彰化銀行機台#彰化銀行為主要機台，適用於一般信用卡及行動支付，而非特定用途的國旅卡。
=玉山銀行機台#玉山銀行機台為特定用途機台，專門適用於國民旅遊卡的刷卡交易。
~德安系統虛擬卡機#德安是前台管理系統，實際刷卡交易仍需透過實體卡機進行。
~任何一台空閒的機台#不同銀行的機台配置有特定用途，國旅卡必須使用指定的玉山銀行機台。
~旅客自行提供的掃碼支付#國民旅遊卡為實體卡交易，且手冊已指定專門的處理機台。

::OP-FO-05-Q03::在信用卡機關班流程中，列印單據的順序至關重要。下列哪一個是正確的單據排列與釘掛順序？ {
~總額報表 $\\rightarrow$ 明細表 $\\rightarrow$ 結帳報表#此順序錯誤，結帳報表應作為第一張覆蓋在最上方。
~明細表 $\\rightarrow$ 結帳報表 $\\rightarrow$ 總額報表#明細表通常是最後一張，不應排在第一位。
=結帳報表 $\\rightarrow$ 總額報表 $\\rightarrow$ 明細表#根據手冊第 $3$ 點之 $3$，卡機結帳三張釘在一起的順序應以此為準。
~結帳報表 $\\rightarrow$ 明細表 $\\rightarrow$ 總額報表#總額報表應排在結帳報表之後、明細表之前。
~視當班刷卡筆數多寡自行調整#為維護稽核便利性，單據順序有嚴格規範，不可隨意調整。
}

::OP-FO-05-Q04::在「德安前台系統」執行關班與開班作業後，下列哪一項是必須執行的後續步驟，以確保系統數據更新？ {
~直接關閉電腦螢幕#僅關閉螢幕無法確保系統數據的登入狀態已更新。
~立即列印所有發票#開關班後應先處理系統登入狀態，而非立即列印發票。
=重新登入系統#手冊第 $4$ 點之 $6$ 明確指出，執行開關班後需重新登入系統以確保權限與班別正確。
~等待五分鐘後再操作#系統不需要閒置等待，而是需要透過重新登入來刷新狀態。
~通知資訊組更新數據#重新登入是現場作業夥伴應自行完成的標準步驟，無需通知資訊組。
}

::OP-FO-05-Q06::在信用卡機結帳流程中，有一個警告提到：「列印單據時，順序不可交換，一旦執行某個動作，資料會全部清除」。請問該動作為何？ {
~取消交易#取消單筆交易不會清除整機的日報資料。
~列印明細#列印明細僅是讀取數據，並不會導致資料消失。
=清機#手冊第 $3$ 點之 $2$ 警告，一旦進行清機，卡機內的當班交易資料將被清除。
~重新開機#重新開機通常會保留當前未結帳的資料，清機才是清除數據的關鍵。
~更換感熱紙#更換耗材不會影響已儲存的電子交易資料。
}

::OP-FO-05-Q07::小明正在列印「交班日報表」，在德安系統中選取出納報表後，他應該輸入什麼資訊來查詢正確的帳務？ {
~旅客姓名與房號#交班日報表是以班別和收入類別為基準，而非單一旅客。
=班別、入賬收入及小分類#根據報表流程 $1$ 之 $4$，查詢交班日報表需輸入班別並選取入賬收入與小分類。
~刷卡機序號#刷卡機序號用於核對實體單據，非德安系統查詢日報表的主要欄位。
~當日天氣概況#天氣資訊與帳務交班日報表的金額核對無關。
~作廢發票張數#雖然需要核對發票，但這不是查詢日報表時的篩選條件。
}

::OP-FO-05-Q08::關於「信用卡黏貼表」的整理規範，下列敘述何者完全正確？ {
~只需將刷卡簽單釘在一起即可，不需黏貼#指導書要求需在黏貼表上貼上簽單並與結帳單釘在一起。
~只需在表單末端簽名，不需寫日期#手冊規定必須簽名加上日期。
=將刷卡簽單貼上，並在兩台卡機結帳單寫上日期及本人簽名#這是第 $3$ 點之 $5$ 明確要求的行政處理流程，確保責任歸屬。
~將所有單據放入現金包中，不需額外黏貼#信用卡簽單有專門的黏貼表處理流程，不同於現金包處理。
~由下班人員代為簽名核實#應由當班本人簽名以示負責。
}
